2's Complement вЂ” The Standard

Why 2's Complement is Universal

To negate: Flip all bits (1's complement), then add 1.
Alternatively: Copy bits from LSB rightward until (and including) the first 1, then flip all remaining bits.
Why it wins: Unique zero. Subtraction done with addition. Same adder circuit handles both signed and unsigned. MSB acts as sign bit naturally.

5.1   Karnaugh Map вЂ” Method & Rules Most Tested

K-Map

A K-map (Karnaugh map) is a visual method for minimizing Boolean expressions. Cells representing adjacent minterms (differing in exactly one variable) are physically adjacent in the map вЂ” this makes grouping easy. Gray code ordering ensures physical adjacency = logical adjacency.

K-Map Grouping Rules

1. Group only 1s (for SOP) or 0s (for POS).
2. Groups must be rectangular and size must be a power of 2: 1, 2, 4, 8, 16.
3. Groups can wrap around edges (the map is toroidal вЂ” top wraps to bottom, left to right).
4. Make groups as large as possible (larger group = fewer variables in the term).
5. Every 1 must be covered by at least one group.
6. Use the minimum number of groups (fewer groups = fewer product terms).
7. Don't-cares (X) can be treated as 0 or 1 вЂ” choose whichever makes larger groups.

5.2   Don't-Care Conditions & POS Minimization GATE Tested

Don't-Care Conditions

Some input combinations may never occur (e.g., BCD digits 1010вЂ“1111 in a BCD circuit) or we don't care what the output is. These are marked X (don't care) in the truth table and K-map. Use them as 1s to enlarge groups for SOP, or as 0s to enlarge groups for POS вЂ” whichever creates larger groupings. Never use a group consisting ONLY of don't-cares вЂ” it doesn't cover any required minterm.

POS Minimization вЂ” Group the 0s

For POS (Product of Sums) minimization:

  • Group the 0s (not the 1s) in the K-map.
  • Each group of 0s gives a sum term (maxterm).
  • Variables that are 0 in the group appear uncomplemented in the sum. Variables that are 1 appear complemented. (Opposite of SOP!)
  • The POS expression is the AND of all sum terms.

6.3   Tri-State Buffers, Parity Circuits & ALU Overview GATE Tested

Tri-State Buffer

A tri-state buffer has three output states: logic 0, logic 1, and high-impedance (Hi-Z). When the enable signal is active, the output follows the input. When disabled, the output is electrically disconnected (Hi-Z) вЂ” allowing multiple buffers to share a bus without conflict. The foundation of all bus architectures.

Parity Generator/Checker

XOR Tree for Parity

XOR of all data bits generates/checks parity. XOR is associative, so an n-input XOR tree takes O(log n) gate levels. Even parity bit = Xв‚Ѓ вЉ• Xв‚‚ вЉ• ... вЉ• Xв‚™. Odd parity bit = complement of even parity. Check: XOR all received bits including parity вЂ” result should be 0 (even) or 1 (odd parity convention).

ALU (Arithmetic Logic Unit)

The ALU performs arithmetic (add, subtract, multiply, compare) and logical (AND, OR, XOR, NOT, shift) operations. It is controlled by function select lines that choose the operation. A 1-bit ALU slice handles one bit and is cascaded to build an n-bit ALU. The carry out of one slice feeds the carry in of the next вЂ” identical to RCA structure. GATE tests: "How many gates are needed for an n-bit ALU?" вЂ” depends on complexity of operations supported.

When E=1 (enabled), the latch is transparent вЂ” Q tracks D continuously. When E=0, Q holds its last value. This level-triggered behavior causes timing hazards in complex circuits вЂ” motivates edge-triggered flip-flops.

7.2   Flip-Flops вЂ” Edge-Triggered GATE Critical

A flip-flop is an edge-triggered device вЂ” it only samples its inputs and updates its output at the clock edge (rising or falling). Between clock edges, the output is stable and the inputs don't affect it. This eliminates the transparency problem of latches.

7.3   Registers, Shift Registers & Counters GATE Tested

Registers

An n-bit register is simply n D flip-flops sharing a common clock. On each rising clock edge, the register loads the n-bit input and holds it. Used as temporary storage in ALUs, CPUs, and pipelines.

Shift Registers

Serial-In Serial-Out (SISO)

Data enters one bit at a time at the input and exits one bit at a time at the output after n clock cycles (n = number of FFs). Used for delay and serial communication.

Serial-In Parallel-Out (SIPO)

Data enters serially, all n bits available in parallel after n clock cycles. Serial-to-parallel conversion. Used in UART receivers.

Parallel-In Serial-Out (PISO)

All n bits loaded in parallel, then shifted out one bit at a time. Parallel-to-serial conversion. Used in UART transmitters.

Parallel-In Parallel-Out (PIPO)

All bits loaded and read simultaneously on clock edge. Standard n-bit register. Used for temporary data storage in data paths.

Counters

Ripple Counter (Asynchronous)

Each FF's output clocks the next FF. Simple but slow вЂ” ripple propagation delay = n Г— FF delay. MSB changes only after all lower bits have settled. Glitches possible. n T flip-flops (with T=1) create a modulo-2вЃї counter.

Synchronous Counter

All FFs share the same clock. All outputs update simultaneously. No ripple delay. Requires more complex logic for T inputs of each stage but much faster and glitch-free. Standard in modern digital design.
