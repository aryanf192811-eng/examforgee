Chapter 01
Propositional Logic — Connectives, Truth Tables & Implication

1.1   Propositions & Connectives Core

A proposition is a declarative statement that is either true or false — never both, never neither. "It is raining" is a proposition. "What time is it?" is not. The truth value of a proposition is either T (true) or F (false).

Compound propositions are built from atomic propositions using logical connectives. Their truth value is determined entirely by the truth values of their components — this is called truth-functional semantics.

Operator precedence (highest to lowest): ¬ → ∧ → ∨ → → → ↔. Parentheses override everything.

GATE Trap — Converse ≠ Contrapositive

p → q and its contrapositive ¬q → ¬p are logically equivalent. The converse q → p and the inverse ¬p → ¬q are logically equivalent to each other, but NOT to the original. GATE exploits this: "Which of the following is equivalent to p → q?" — only the contrapositive is correct.

Memory Aid for Connectives

∧ (AND): both must be T → only (T,T) → T. ∨ (OR): only (F,F) → F. ⊕ (XOR): same → F, different → T. → (implication): only (T,F) → F. ↔ (biconditional): same → T, different → F. XOR and ↔ are exact opposites — p⊕q = ¬(p↔q).

1.3   Tautology, Contradiction & Contingency GATE Tested

Tautology

True for ALL assignments of truth values. Example: p ∨ ¬p (law of excluded middle). All rows in truth table are T.

Contradiction (Unsatisfiable)

False for ALL assignments. Example: p ∧ ¬p. All rows in truth table are F. Also called a "fallacy."

Contingency

True for some, false for others. Example: p → q. Neither tautology nor contradiction. Most propositions are contingencies.

Satisfiable

True for AT LEAST ONE assignment. Includes tautologies and contingencies. NOT satisfiable = contradiction.

Classify: (p → q) ∧ (q → p)

This is p ↔ q — a contingency. True when p and q have the same value, false otherwise.

Classify: (p → q) ∨ (q → p)

This is a tautology. Proof: if p=T, then p→q is T when q=T, and q→p=T always when p=T. If p=F, then p→q=T always. Every row is T.

Logical Equivalences & Normal Forms

2.1   Logical Equivalence Laws GATE Tested

Two propositions P and Q are logically equivalent (P ≡ Q) if they have identical truth values for every possible assignment. Equivalently, P ↔ Q is a tautology.

De Morgan's — Apply to Both AND and OR

De Morgan's law: negate the whole expression, flip AND↔OR, negate each part. ¬(p∧q) = ¬p∨¬q. ¬(p∨q) = ¬p∧¬q. This extends to n variables: ¬(p₁∧p₂∧...∧pₙ) = ¬p₁∨¬p₂∨...∨¬pₙ. GATE applies De Morgan's frequently in simplification questions.

2.2   Normal Forms — CNF and DNF GATE Tested

Literal

A variable or its negation: p, ¬p, q, ¬q. The building block of normal forms.

Conjunctive Normal Form (CNF)

A conjunction (AND) of clauses, where each clause is a disjunction (OR) of literals.
Form: (l₁ ∨ l₂ ∨ ...) ∧ (l₃ ∨ l₄ ∨ ...) ∧ ...
Also called: Product of Sums (POS). Every proposition has a CNF.

Disjunctive Normal Form (DNF)

A disjunction (OR) of clauses, where each clause is a conjunction (AND) of literals.
Form: (l₁ ∧ l₂ ∧ ...) ∨ (l₃ ∧ l₄ ∧ ...) ∨ ...
Also called: Sum of Products (SOP). Every proposition has a DNF.

Convert p → (q ∧ r) to CNF and DNF:

Original: p → (q ∧ r) Step 1 — Eliminate →: ¬p ∨ (q ∧ r) Step 2 — Distribute ∨ over ∧ (for CNF): (¬p ∨ q) ∧ (¬p ∨ r) ← CNF ✓ For DNF — push negations in, use distributivity: ¬p ∨ (q ∧ r) ← already DNF ✓ (¬p is a clause with one literal; (q∧r) is a clause with two)

Canonical Forms

The canonical CNF (or PCNF — Principal CNF) uses maxterms — one clause per row in the truth table where the formula is FALSE. The canonical DNF (or PDNF) uses minterms — one clause per row where the formula is TRUE.

From truth table of p → q (rows: TT→T, TF→F, FT→T, FF→T):

False only at row (T,F): minterm for FALSE = (p ∧ ¬q). So PCNF has one maxterm: (¬p ∨ q). That IS p → q itself — already in canonical CNF.

True at rows (T,T), (F,T), (F,F): PDNF = (p∧q) ∨ (¬p∧q) ∨ (¬p∧¬q).

CNF and SAT — Why This Matters Beyond GATE

The SAT problem (Boolean Satisfiability) — "is there an assignment making a CNF formula true?" — is the canonical NP-complete problem (Cook-Levin theorem). Every problem in NP can be polynomial-time reduced to SAT. This is the theoretical reason why CNF appears everywhere in computer science — compilers, hardware verification, AI planning.

A set of connectives is functionally complete if every Boolean function can be expressed using only the connectives in that set.

NAND and NOR Are Universal Gates

NAND (↑) and NOR (↓) are each individually functionally complete — this is why they are called "universal gates" in digital circuits. You can build any circuit using only NAND gates, or only NOR gates. This is a hardware design principle that comes directly from functional completeness.

Chapter 03
Predicate Logic & Quantifiers

3.1   Predicates, Variables & Quantifiers Core

Propositional logic treats statements as atomic. Predicate logic (first-order logic) looks inside statements — it expresses properties of objects and relationships between them. This is the language of mathematics.

Predicate

A predicate P(x) is a propositional function — a statement with a variable that becomes a proposition when the variable is assigned a value from the domain. Example: P(x) = "x is prime." P(7) is true. P(6) is false.

Universal Quantifier — ∀

∀x P(x) means "For ALL x in the domain, P(x) is true." It is equivalent to the conjunction of P(x) over all domain elements: P(x₁) ∧ P(x₂) ∧ P(x₃) ∧ ...

Existential Quantifier — ∃

∃x P(x) means "There EXISTS at least one x in the domain such that P(x) is true." It is equivalent to the disjunction: P(x₁) ∨ P(x₂) ∨ P(x₃) ∨ ...

∀x P(x)P(x) true for every x in domainThere exists some x where P(x) is false (counterexample)
∃x P(x)P(x) true for at least one xP(x) false for every x in domain
∀x ¬P(x)P(x) false for every xThere exists some x where P(x) is true
∃x ¬P(x)P(x) false for at least one xP(x) true for every x

3.2   Negation of Quantifiers — De Morgan's for Predicate Logic Most Tested

Quantifier Negation Laws

¬(∀x P(x)) ≡ ∃x ¬P(x)
¬(∃x P(x)) ≡ ∀x ¬P(x)

These are the predicate-logic analog of De Morgan's laws. Negating a universal statement gives an existential one, and vice versa. To disprove "all X are Y," find one X that is not Y. To disprove "some X is Y," show that no X is Y.

Negate: "Every student in this class has studied calculus."

Formal: ∀x (Student(x) → StudiedCalculus(x))

Negation: ∃x ¬(Student(x) → StudiedCalculus(x)) ≡ ∃x (Student(x) ∧ ¬StudiedCalculus(x))

English: "There is some student in this class who has NOT studied calculus."

Note: negation of (p → q) = p ∧ ¬q — this is the only case where implication is false.

Nested Quantifiers — Order Matters

∀x∃y ≠ ∃y∀x — The Most Tested Quantifier Trap

∀x ∃y P(x,y): For every x, there exists some y (possibly different for each x) such that P(x,y).
∃y ∀x P(x,y): There exists one specific y that works for ALL x simultaneously.

∃y∀x implies ∀x∃y (the single y works for every x, so in particular for each x individually). But ∀x∃y does NOT imply ∃y∀x. The second is strictly stronger.

Example: "Every person has a mother" = ∀x∃y Mother(y,x). "There is someone who is everyone's mother" = ∃y∀x Mother(y,x). Clearly the first is true and the second is false.

StatementReadingStrength
∀x ∀y P(x,y)P holds for every pair (x,y)Strongest
∀x ∃y P(x,y)For each x, some y works (y may depend on x)Strong
∃x ∀y P(x,y)Some specific x works for all yModerate
∃x ∃y P(x,y)Some pair (x,y) satisfies PWeakest

Chapter 04
Proof Techniques — Direct, Contrapositive, Contradiction & Induction

4.1   Methods of Proof GATE Tested

Direct Proof

To prove p → q: assume p is true, use axioms, definitions, and previously proven theorems to derive q. Most straightforward method.

Prove: if n is odd, then n² is odd.

Assume n is odd. Then n = 2k+1 for some integer k. So n² = (2k+1)² = 4k²+4k+1 = 2(2k²+2k)+1. This is of the form 2m+1, hence odd. ∎

Proof by Contrapositive

To prove p → q: equivalently prove ¬q → ¬p. Often easier when the conclusion q is simpler to negate than to directly prove.

Prove: if n² is even, then n is even.

Contrapositive: if n is odd, then n² is odd. (Proved directly above.) ∎

The direct proof would require working with "n² is even" which requires extracting a square root — much harder. Contrapositive is the clean path.

Proof by Contradiction

To prove p: assume ¬p, derive a contradiction (something known to be false, like r ∧ ¬r). Conclude p must be true. Also called reductio ad absurdum.

Prove: √2 is irrational.

Assume √2 is rational. Then √2 = a/b where a,b are integers, b≠0, gcd(a,b)=1 (fully reduced). Then 2 = a²/b², so a² = 2b². → a² is even → a is even (from contrapositive proved above). Let a = 2k. Then (2k)² = 2b², so 4k² = 2b², so b² = 2k². → b² is even → b is even. But then gcd(a,b) ≥ 2. Contradicts gcd(a,b)=1. ∎

Mathematical Induction

Principle of Mathematical Induction

To prove ∀n ≥ n₀, P(n):
Base case: Prove P(n₀) is true.
Inductive step: Assume P(k) is true for some arbitrary k ≥ n₀ (inductive hypothesis). Prove P(k+1) is true.
Conclusion: P(n) is true for all n ≥ n₀.

Prove: 1 + 2 + 3 + ... + n = n(n+1)/2

Base case: n=1. LHS = 1. RHS = 1(2)/2 = 1. ✓ Inductive step: Assume 1+2+...+k = k(k+1)/2. Prove 1+2+...+k+(k+1) = (k+1)(k+2)/2. LHS = k(k+1)/2 + (k+1) = (k+1)[k/2 + 1] = (k+1)(k+2)/2 = RHS. ✓ ∎

Strong Induction

Strong (Complete) Induction

To prove ∀n ≥ n₀, P(n):
Base case(s): Prove P(n₀) (and possibly a few more).
Inductive step: Assume P(n₀), P(n₀+1), ..., P(k) are ALL true. Prove P(k+1).
Used when P(k+1) depends on multiple prior values, not just P(k). Example: Fibonacci, prime factorization proofs.

Structural Induction

Induction generalized to recursively defined structures (trees, lists, strings). Base case: smallest elements. Inductive step: if property holds for parts, show it holds for the whole. Used in proofs about parse trees, expression evaluation, and BST properties.

Chapter 06
Relations — Properties & Closures

6.1   Relations — Definition & Representation Core

Binary Relation

A binary relation R from set A to set B is a subset of A × B. If (a,b) ∈ R we write aRb ("a is related to b"). A relation on A is a subset of A × A (from A to itself).

A relation on a set A with n elements can be represented as:

  • Set of pairs: R = {(1,2), (2,3), (3,1)} ⊆ {1,2,3} × {1,2,3}
  • Matrix representation: n×n boolean matrix M where M[i][j] = 1 iff (i,j) ∈ R
  • Directed graph: nodes = elements, directed edge i→j iff (i,j) ∈ R

Symmetric vs Antisymmetric — NOT Opposites

A relation CAN be both symmetric and antisymmetric — example: the equality relation = on any set. (a=b ∧ b=a → a=b trivially; and (a,b)∈R ∧ (b,a)∈R means a=b=a, so symmetric.) A relation can also be neither symmetric nor antisymmetric — example: {(1,2),(2,1),(1,3)} is not symmetric (no (3,1)) and not antisymmetric ((1,2) and (2,1) with 1≠2).

Equivalence Relations

Equivalence Relation

A relation R on A is an equivalence relation if it is reflexive, symmetric, and transitive. An equivalence relation partitions A into disjoint equivalence classes [a] = {b ∈ A | aRb}.

Congruence modulo n: a ≡ b (mod n) iff n | (a−b). This is an equivalence relation on ℤ. The equivalence classes are {0,n,2n,...}, {1,n+1,2n+1,...}, ..., {n−1,2n−1,...} — exactly n classes.

Partial Order Relations (POSET)

Partial Order

A relation R on A is a partial order if it is reflexive, antisymmetric, and transitive. The pair (A, R) is a partially ordered set (POSET). Example: (P(S), ⊆) — power set ordered by subset inclusion.

Total Order (Linear Order)

A partial order where every pair of elements is comparable: ∀a,b: (a,b)∈R OR (b,a)∈R. Example: (ℤ, ≤) — any two integers are comparable.

6.3   Relation Composition & Closures GATE Tested

Composition of Relations

If R ⊆ A×B and S ⊆ B×C, then R∘S = {(a,c) | ∃b: (a,b)∈R ∧ (b,c)∈S}. Also written S∘R in some books — check convention. Represented in matrix form as boolean matrix multiplication: M_{R∘S} = M_R · M_S (boolean).

Transitive Closure

The transitive closure R⁺ of R is the smallest transitive relation containing R. R⁺ = R ∪ R² ∪ R³ ∪ ... (where Rⁿ = R composed with itself n times). The matrix of R⁺ = M_R ∨ M_R² ∨ M_R³ ∨ ... (boolean OR).

Computed efficiently using Warshall's algorithm in O(n³).

Closure TypeAdd the minimum pairs to make R...Method
Reflexive closureReflexiveAdd (a,a) for all a ∈ A: R ∪ Δ (diagonal)

There Is No "Antisymmetric Closure"

You can always add pairs to make a relation reflexive, symmetric, or transitive — the added pairs only make the relation larger. But making a relation antisymmetric requires removing pairs, not adding them. Since closures only add pairs, there is no antisymmetric closure operation.

Chapter 07
Functions — Types, Composition & Cardinality

7.1   Functions — Definition & Types GATE Critical

Function

A function f: A → B is a relation from A to B where every element of A is related to exactly one element of B. A is the domain, B is the codomain, and the set of actual output values is the range (or image): f(A) = {f(a) | a ∈ A} ⊆ B.

Injective (One-to-One)

∀a₁,a₂: f(a₁)=f(a₂) → a₁=a₂.
Different inputs → different outputs. No two elements map to the same output. Range size = domain size.

Surjective (Onto)

∀b ∈ B, ∃a ∈ A: f(a) = b.
Every element of the codomain has at least one preimage. Range = Codomain.

Bijective (One-to-One Correspondence)

Both injective AND surjective. Perfect pairing — every element of B has exactly one preimage. Invertible.

None of the above

Not injective (two inputs → same output) AND/OR not surjective (some output never reached). Most general case.

f: ℤ → ℤ, f(n) = 2n

  • Injective? Yes — f(a)=f(b) means 2a=2b means a=b.
  • Surjective? No — f(n)=3 has no solution (3 is odd, 2n is always even).
  • Bijective? No (not surjective).

f: ℤ → ℤ, f(n) = n+1

  • Injective? Yes. Surjective? Yes (f(n-1)=n for any n). Bijective? Yes.

How many functions f: {1,2,3} → {a,b,c,d} are injective?

P(4,3) = 4!/(4−3)! = 4!/1! = 24. First element: 4 choices. Second: 3 remaining. Third: 2 remaining. Total: 4×3×2 = 24.

How many functions f: {1,2,3,4} → {a,b} are surjective?

Total = 2⁴ = 16. Subtract: all map to {a} = 1, all map to {b} = 1. By inclusion-exclusion: 16 − 1 − 1 = 14.

Bijection and Cardinality

Two sets A and B have the same cardinality (|A| = |B|) if and only if there exists a bijection between them. This extends to infinite sets: ℕ and ℤ have the same cardinality (bijection exists). ℕ and ℝ do NOT (Cantor's diagonal argument — ℝ is uncountable). This is the foundation of infinite set theory.

7.3   Composition & Inverse GATE Tested

Composition

(g∘f)(x) = g(f(x)). Apply f first, then g. Note order: g∘f means "f first." Composition is associative: (h∘g)∘f = h∘(g∘f). Not commutative in general: g∘f ≠ f∘g.

Inverse Function

A function f: A→B has an inverse f⁻¹: B→A if and only if f is bijective. f⁻¹(b) = a where f(a)=b. f∘f⁻¹ = identity on B, f⁻¹∘f = identity on A.

8.1   All the Traps in One Place

2Converse ≡ Contrapositivep→q ≡ ¬q→¬p (contrapositive). Converse q→p is NOT equivalent. Students confuse these every year. Contrapositive: flip AND negate both. Converse: just flip.
3∀x∃y ≡ ∃y∀xThese are NOT equivalent. ∃y∀x is strictly stronger — one y must work for all x. ∀x∃y allows y to depend on x. Swapping quantifiers of the same type is fine; swapping ∀ and ∃ changes meaning.
4Affirming the consequentp→q, q ⊢ p is INVALID. The correct rule is Modus Ponens: p→q, p ⊢ q. Having q doesn't tell you p was the cause.

6Symmetric and Antisymmetric are oppositesThey're not. The equality relation is BOTH symmetric and antisymmetric. A relation can also be neither. Symmetric: (a,b)∈R → (b,a)∈R. Antisymmetric: (a,b)∈R ∧ (b,a)∈R → a=b.

8.3   GATE Question Patterns

  • Truth table evaluation: Given a formula, find number of assignments that make it true. Count T rows — know operator precedence cold.
  • Logical equivalence: "Which of the following is equivalent to p→q?" Options include converse, inverse, contrapositive, and unrelated. Only contrapositive is correct.
  • Tautology check: Given a formula, determine if it's a tautology (all T), contradiction (all F), or contingency. Often done algebraically — use equivalence laws instead of full truth table.
  • Quantifier negation: Given a statement with nested quantifiers, find its negation. Push ¬ inward: ¬∀→∃, ¬∃→∀, ¬(P→Q)→P∧¬Q.
  • Relation properties: Given a relation as a set of pairs or matrix, determine which properties hold (reflexive, symmetric, transitive, etc.). Check all 5 properties systematically.
  • Equivalence relation / POSET: Identify which properties a relation satisfies, classify it as equivalence relation or partial order. Find equivalence classes.
  • Function counting NAT: "How many injective functions from a 3-element set to a 5-element set?" = P(5,3) = 60. Direct formula application, 1–2 marks.
  • Set cardinality: Given |A|=m, |B|=n, find |P(A)|, |A×B|, |P(P(A))|. Apply formulas, 1 mark.
  • Proof validity: Given an argument, identify if it uses a valid inference rule or a fallacy (affirming the consequent, denying the antecedent).
  • Inclusion-exclusion: Given overlapping set sizes, find union or elements in exactly k sets. 2-mark calculation question.

Number Theory — GCD, Modular Arithmetic & Fermat

9.1   GCD, LCM & Euclidean Algorithm GATE Critical

GCD — Greatest Common Divisor

gcd(a, b) = largest positive integer that divides both a and b. Euclidean Algorithm: gcd(a, b) = gcd(b, a mod b). Repeat until remainder = 0; the non-zero argument at that point is the GCD. Time complexity: O(log min(a,b)).

Compute gcd(252, 105):

gcd(252, 105): 252 = 2×105 + 42 gcd(105, 42): 105 = 2×42 + 21 gcd(42, 21): 42 = 2×21 + 0 → GCD = 21 lcm(252, 105) = 252×105 / 21 = 26460 / 21 = 1260
Bézout's Identity

For integers a and b with gcd(a,b) = d, there exist integers x and y such that ax + by = d. These x, y can be found by the Extended Euclidean Algorithm (back-substituting through the Euclidean steps). The equation ax + by = c has integer solutions iff d | c.

9.2   Modular Arithmetic GATE Critical

Congruence and Modular Properties

a ≡ b (mod n) means n | (a − b). Equivalence relation on ℤ.
If a ≡ b (mod n) and c ≡ d (mod n) then:
a + c ≡ b + d (mod n)   (addition preserved)
a × c ≡ b × d (mod n)   (multiplication preserved)
aᵏ ≡ bᵏ (mod n)         (powers preserved)

Modular Inverse

The modular inverse of a mod n is x such that ax ≡ 1 (mod n). It exists iff gcd(a, n) = 1 (a and n are coprime). Found using Extended Euclidean Algorithm or by Fermat's Little Theorem when n is prime: a⁻¹ ≡ a^(p−2) (mod p).

Find 3⁻¹ mod 7:

Need x such that 3x ≡ 1 (mod 7). Test: 3×1=3, 3×2=6, 3×3=9≡2, 3×4=12≡5, 3×5=15≡1 ✓ So 3⁻¹ ≡ 5 (mod 7). Via Fermat: 3⁻¹ ≡ 3^(7−2) = 3⁵ = 243 ≡ 243 mod 7 243 = 34×7 + 5 → 243 ≡ 5 (mod 7) ✓

Modular Exponentiation — Fast Power

Repeated Squaring

Compute aⁿ mod m in O(log n) multiplications:
Write n in binary. Start with result = 1, base = a mod m.
For each bit of n (from LSB): if bit = 1, multiply result by base (mod m). Square base (mod m). Continue.

Compute 3⁴⁴ mod 5:

44 = 101100₂ 3¹ mod 5 = 3 3² mod 5 = 9 mod 5 = 4 3⁴ mod 5 = 4² mod 5 = 16 mod 5 = 1 3⁸ mod 5 = 1² mod 5 = 1 3¹⁶ mod 5 = 1 3³² mod 5 = 1 44 = 32 + 8 + 4: 3⁴⁴ = 3³² × 3⁸ × 3⁴ mod 5 = 1 × 1 × 1 = 1 mod 5

9.3   Fermat's Little Theorem & Euler's Totient GATE Tested

Fermat's Little Theorem

If p is prime and gcd(a, p) = 1 (a not divisible by p):
aᵖ⁻¹ ≡ 1 (mod p)
Equivalently: aᵖ ≡ a (mod p) for all a.
Use: Reduces large exponents modulo (p−1) when working mod p. Finds modular inverses: a⁻¹ ≡ aᵖ⁻² (mod p).

Euler's Totient Function φ(n)

φ(n) = count of integers in [1, n] that are coprime to n (gcd = 1).
φ(1) = 1. For prime p: φ(p) = p − 1. For prime power: φ(pᵏ) = pᵏ − pᵏ⁻¹ = pᵏ⁻¹(p−1).
Multiplicative: if gcd(m,n)=1: φ(mn) = φ(m)·φ(n).
For n = p₁^a₁ · p₂^a₂ · ... : φ(n) = n · (1−1/p₁) · (1−1/p₂) · ...
Euler's Theorem: If gcd(a,n)=1: a^φ(n) ≡ 1 (mod n). Generalizes Fermat.

Compute φ(36) and φ(100):

φ(36): 36 = 2²×3². φ(36) = 36×(1−1/2)×(1−1/3) = 36×1/2×2/3 = 12 φ(100): 100 = 2²×5². φ(100) = 100×(1−1/2)×(1−1/5) = 100×1/2×4/5 = 40
Chinese Remainder Theorem (CRT)

If n₁, n₂, ..., nₖ are pairwise coprime (gcd(nᵢ, nⱼ)=1 for i≠j), then the system:
x ≡ a₁ (mod n₁), x ≡ a₂ (mod n₂), ..., x ≡ aₖ (mod nₖ)
has a unique solution modulo N = n₁·n₂·...·nₖ.

Find x: x ≡ 2 (mod 3), x ≡ 3 (mod 5), x ≡ 2 (mod 7):

N = 3×5×7 = 105 N₁ = 35, N₂ = 21, N₃ = 15 Find inverses: 35y₁ ≡ 1 (mod 3) → 2y₁ ≡ 1 (mod 3) → y₁ = 2 21y₂ ≡ 1 (mod 5) → 1y₂ ≡ 1 (mod 5) → y₂ = 1 15y₃ ≡ 1 (mod 7) → 1y₃ ≡ 1 (mod 7) → y₃ = 1 x = (2×35×2 + 3×21×1 + 2×15×1) mod 105 = (140 + 63 + 30) mod 105 = 233 mod 105 = 23 Verify: 23 mod 3 = 2 ✓, 23 mod 5 = 3 ✓, 23 mod 7 = 2 ✓
