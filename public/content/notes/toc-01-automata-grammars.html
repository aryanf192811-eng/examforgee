Chapter 01
Languages & Automata Foundations

1.1   Alphabets, Strings & Languages Core

Theory of Computation studies what problems computers can and cannot solve, and how efficiently. The foundation is a precise mathematical vocabulary for describing what a computation operates on.

Alphabet (Σ)

A finite, non-empty set of symbols. Example: Σ = {0, 1} (binary alphabet), Σ = {a, b, c} (ternary). Every formal language is defined over some alphabet.

String (Word)

A finite sequence of symbols from Σ. The empty string ε contains zero symbols. The length of string w is written |w|. |ε| = 0.

Language (L)

A set of strings over Σ. A language can be finite or infinite. L ⊆ Σ*. Special cases: ∅ (empty language, contains no strings) and {ε} (language containing only the empty string — distinct from ∅).

∅ ≠ {ε} — The Most Confused Pair in TOC

∅ is the empty language — it contains no strings at all. {ε} is a language containing exactly one string: the empty string. |∅| = 0 (zero strings). |{ε}| = 1 (one string, which happens to be empty). This distinction is tested directly in GATE.

1.2   What Is a Finite Automaton? Core

A finite automaton is a mathematical model of a machine with finite memory. It reads an input string one symbol at a time, transitions between a finite set of states based on each symbol, and at the end decides: accept (the string is in the language) or reject (it is not).

The class of languages recognized by finite automata is exactly the class of Regular Languages. This is a fundamental equivalence — it says: a language can be described by a DFA if and only if it is regular.

DFA

Deterministic Finite Automaton. From every state, on every input symbol, there is exactly one transition. Most concrete model.

NFA

Nondeterministic FA. From a state, on a symbol, there can be zero, one, or many transitions. Easier to design — equivalent power to DFA.

ε-NFA

NFA with ε-transitions — can change state without reading any input. Still equivalent in power to DFA and NFA.

PDA

Pushdown Automaton. FA + stack memory. Recognizes Context-Free Languages — strictly more powerful than finite automata.

Chapter 02
DFA — Deterministic Finite Automata

2.1   Formal Definition Core

DFA — 5-Tuple Definition

A DFA is a 5-tuple M = (Q, Σ, δ, q₀, F) where:
Q — finite set of states
Σ — input alphabet (finite set of symbols)
δ: Q × Σ → Q — transition function (deterministic: exactly one next state)
q₀ ∈ Q — start state (initial state)
F ⊆ Q — set of accept states (final states)

The DFA processes input by starting in q₀ and applying δ repeatedly. After reading the entire input string, if the current state is in F, the string is accepted. Otherwise it is rejected.

The extended transition function δ*: Q × Σ* → Q generalizes δ to handle entire strings: δ*(q, ε) = q and δ*(q, wa) = δ(δ*(q, w), a).

The language accepted by M is: L(M) = {w ∈ Σ* | δ*(q₀, w) ∈ F}.

ε is accepted (0 zeros = even). "100" has one 0 → rejected. "1001" has two 0s → accepted.

DFA Design Strategy — Think in States as Memory

Each state encodes a "memory" about the input seen so far that is relevant to acceptance. Ask: "what do I need to remember about the input read so far to decide acceptance?" The answer gives you your states. For "ends in ab": I need to remember the last 1-2 characters → 3 states. For "even number of 0s": I need to remember parity → 2 states.

2.3   DFA Minimization — Table Filling Algorithm GATE Tested

A minimal DFA is the unique smallest DFA (fewest states) for a given regular language. Two states are distinguishable if there exists some string that leads one to an accept state and the other to a reject state.

Table-Filling (Myhill-Nerode) Algorithm

Step 1: Mark all pairs (accept, non-accept) as distinguishable — they differ on ε.
Step 2: For each unmarked pair (p, q) and each symbol a: if (δ(p,a), δ(q,a)) is already marked, mark (p, q).
Step 3: Repeat step 2 until no new pairs are marked.
Step 4: All unmarked pairs are equivalent — merge them into a single state.

Myhill-Nerode Theorem — The Deep Result

A language L is regular if and only if it has a finite number of equivalence classes under the relation: x ≡_L y iff for all z, xz ∈ L ↔ yz ∈ L. The number of equivalence classes equals the number of states in the minimal DFA. GATE uses this theorem to prove a language is NOT regular (infinite classes → no finite DFA possible).

GATE NAT — Minimum States

"What is the minimum number of states in a DFA accepting all strings over {0,1} that contain the substring '01'?" — Think: you need to track: (1) haven't seen '0' yet, (2) seen '0' but not '01', (3) seen '01' (accept, dead state). Answer: 3 states. These minimum-state questions are extremely common 1-mark NAT questions.

Chapter 03
NFA & ε-NFA — Nondeterminism

3.1   NFA — Formal Definition & Semantics Core

NFA — 5-Tuple Definition

An NFA is M = (Q, Σ, δ, q₀, F) where δ: Q × Σ → 2^Q (power set of Q).
Unlike DFA where δ returns a single state, NFA's δ returns a set of states (possibly empty, possibly multiple).

Nondeterminism means: at each step, the NFA can be in multiple states simultaneously. Think of it as exploring all possible computation paths in parallel. An NFA accepts a string if at least one of those parallel computation paths ends in an accept state.

If δ(q, a) = ∅ for some state q and input a, that computation path dies (not a rejection of the whole string — other parallel paths may still lead to acceptance).

NFA for strings ending in 'ab' (simpler than the DFA):

States: q0 (start), q1 (read 'a'), q2 (read 'ab') ← ACCEPT δ(q0, a) = {q0, q1} ← nondeterministically: stay OR start matching 'ab' δ(q0, b) = {q0} ← stay in q0 δ(q1, b) = {q2} ← complete the 'ab' suffix δ(q2, a) = ∅ ← dead (but other paths may survive) δ(q2, b) = ∅ ← dead Trace "aab": Path 1: q0→(a)→q0→(a)→q0→(b)→q0 NOT in F Path 2: q0→(a)→q0→(a)→q1→(b)→q2 IN F ✓ → ACCEPTED

The NFA is much simpler to design than the equivalent DFA — this is the practical value of nondeterminism.

3.2   NFA → DFA Conversion: Subset Construction Most Tested

Subset Construction Algorithm

Every NFA can be converted to an equivalent DFA. The DFA's states are subsets of the NFA's states. If the NFA has n states, the DFA has at most 2ⁿ states (but usually far fewer in practice).

  1. 1
    Start state of DFA = {q₀} (singleton set containing NFA's start state).
  2. 2
    For each DFA state S (which is a set of NFA states) and each symbol a: compute δ_DFA(S, a) = ∪_{q∈S} δ_NFA(q, a). This is the set of all NFA states reachable from any state in S on input a.
  3. 3
    If a new set appears that hasn't been a DFA state yet, add it.
  4. 4
    A DFA state is an accept state if it contains at least one NFA accept state.
  5. 5
    The empty set ∅ becomes a dead/trap state (non-accepting, all transitions loop back to ∅).

NFA → DFA conversion for the 'ab'-ending NFA:

NFA states: {q0, q1, q2} Accept: {q2} δ_NFA(q0,a)={q0,q1} δ_NFA(q0,b)={q0} δ_NFA(q1,a)=∅ δ_NFA(q1,b)={q2} δ_NFA(q2,a)=∅ δ_NFA(q2,b)=∅ Subset Construction: DFA start = {q0} From {q0}: on a → {q0,q1} on b → {q0} From {q0,q1}: on a → {q0,q1} on b → {q0,q2} ← contains q2 → ACCEPT From {q0,q2}: on a → {q0,q1} on b → {q0} DFA states: {q0}, {q0,q1}, {q0,q2} → only 3 states (not 2³=8)

Notice: the DFA has exactly 3 states, matching our original DFA design. Subset construction found the minimal structure automatically.

State Explosion — Worst Case 2ⁿ

For some languages, NFA→DFA conversion genuinely requires exponentially more states. An NFA with n states can require a DFA with exactly 2ⁿ states. GATE asks: "An NFA with 3 states — what is the maximum number of states in the equivalent minimal DFA?" Answer: 2³ = 8. But for most practical languages it's much less.

3.3   ε-NFA & ε-Closure GATE Tested

ε-NFA

An NFA where δ: Q × (Σ ∪ {ε}) → 2^Q. Transitions on ε (epsilon) change state without consuming any input symbol. Useful for composing automata — connect them with ε-transitions.

ε-closure(q)

The set of all states reachable from state q using zero or more ε-transitions alone. Always includes q itself. ε-closure(q) = {q} ∪ {all states reachable via ε-transitions from q}.

Computing ε-closure: States q0, q1, q2, q3 with ε-transitions: q0→q1, q1→q2, q2→q3 (all ε).

ε-closure(q0) = {q0, q1, q2, q3} — following all ε-chains from q0.

ε-closure(q2) = {q2, q3} — only q2 and q3 reachable by ε from q2.

ε-NFA → DFA Conversion

Same subset construction, but DFA states are ε-closures of NFA state sets:

  1. 1
    DFA start state = ε-closure(q₀).
  2. 2
    For DFA state S and symbol a: compute ε-closure(∪_{q∈S} δ(q,a)).
  3. 3
    Accept state if contains any NFA accept state.
ModelTransition FunctionPowerConversion to DFA
DFAδ: Q×Σ → QRegular LanguagesAlready a DFA
NFAδ: Q×Σ → 2^QRegular LanguagesSubset construction (≤2ⁿ states)
ε-NFAδ: Q×(Σ∪{ε}) → 2^QRegular Languagesε-closure + subset construction
All Three Recognize Exactly the Same Class

DFA = NFA = ε-NFA in terms of language recognition power. All three recognize exactly the regular languages. None is more "powerful" than another. The differences are only in ease of design — NFA and ε-NFA are often more compact and intuitive. GATE frequently asks "which of these recognizes a non-regular language?" to test if you confuse NFA with PDA.

Regular Expressions & Conversions

4.1   Regular Expressions — Syntax & Semantics Core

Inductive Definition of Regular Expressions over Σ

Base cases: ∅ is a RE (language: ∅). ε is a RE (language: {ε}). For each a ∈ Σ, 'a' is a RE (language: {a}).

Inductive cases: If R and S are REs over Σ, then:
(R + S) or (R | S) — Union: L(R) ∪ L(S)
(RS) — Concatenation: L(R)·L(S) = {xy | x∈L(R), y∈L(S)}
(R*) — Kleene star: L(R)* = L(R)⁰ ∪ L(R)¹ ∪ L(R)² ∪ ... (zero or more repetitions)

Precedence: * (highest) → concatenation → + (lowest). Use parentheses to override.

Regular ExpressionLanguage Described
a*{ε, a, aa, aaa, ...} — zero or more a's
a⁺{a, aa, aaa, ...} — one or more a's. Note: a⁺ = aa*
(a+b)*All strings over {a,b} including ε — same as Σ*
a*b*Strings of zero or more a's followed by zero or more b's: {ε, a, b, aa, ab, bb, aab, ...}
(ab)*{ε, ab, abab, ababab, ...} — zero or more copies of 'ab'
a*ba*Strings over {a,b} with exactly one b
(a+b)*abbAll strings ending in 'abb'
∅*{ε} — Kleene star of empty language is {ε}
∅* = {ε} — Counterintuitive but True

The Kleene star includes zero repetitions (the empty string ε). So ∅* = ∅⁰ ∪ ∅¹ ∪ ... = {ε} ∪ ∅ ∪ ∅ ∪ ... = {ε}. GATE tests this as a 1-mark trap.

5.1   Closure Properties GATE Tested

If L₁ and L₂ are regular languages, the following operations produce regular languages:

Complement Construction — Careful with NFA

Complementing a DFA is easy: swap accept ↔ non-accept states. The resulting DFA accepts exactly the complement. But you cannot complement an NFA this way — an NFA can be in multiple states, so swapping accept/non-accept states gives wrong results. Always convert NFA → DFA first, then complement. GATE has tested this exact distinction.

5.2   Pumping Lemma for Regular Languages Most Tested

Pumping Lemma — Statement

If L is a regular language, then there exists a pumping length p ≥ 1 such that for every string w ∈ L with |w| ≥ p, there exists a decomposition w = xyz satisfying:
(1) |xy| ≤ p    (2) |y| ≥ 1    (3) for all i ≥ 0, xy^i z ∈ L
The string y (the "pump") can be repeated any number of times (including 0) and the result must stay in L.

The pumping lemma is used to prove languages are NOT regular. The proof strategy is always a contradiction argument: assume L is regular (so the lemma must hold), show that it can't hold for some specific string, conclude L is not regular.

Proof Strategy — The 6-Step Method

  1. 1
    Assume for contradiction that L is regular. Let p be the pumping length.
  2. 2
    Choose a specific string w ∈ L with |w| ≥ p. You choose w — choose it cleverly to force a contradiction.
  3. 3
    The pumping lemma says w = xyz with |xy| ≤ p, |y| ≥ 1. The adversary chooses the split — you must handle all possible splits.
  4. 4
    Reason about what y must look like given |xy| ≤ p.
  5. 5
    Find a specific i ≥ 0 such that xy^i z ∉ L. This is your contradiction.
  6. 6
    Conclude L is not regular.

Prove L = {aⁿbⁿ | n ≥ 0} is not regular.

Step 1: Assume L is regular, p is the pumping length. Step 2: Choose w = aᵖbᵖ ∈ L (has p a's followed by p b's). |w| = 2p ≥ p. ✓ Step 3: By pumping lemma, w = xyz, |xy| ≤ p, |y| ≥ 1. Step 4: Since |xy| ≤ p and the first p characters of w are all a's, xy is entirely within the 'a' region. Therefore y = aᵏ for some k ≥ 1. So w = aᵖ⁻ᵏ · aᵏ · bᵖ (i.e., x = aᵖ⁻ᵏ, y = aᵏ, z = bᵖ) Step 5: Pump i=2: xy²z = aᵖ⁻ᵏ · a²ᵏ · bᵖ = aᵖ⁺ᵏ bᵖ Number of a's = p+k > p = number of b's. → xy²z ∉ L (unequal counts). CONTRADICTION. Step 6: L = {aⁿbⁿ} is not regular. ∎
Common Mistakes in Pumping Lemma Proofs

Mistake 1 — Choosing y yourself: The pumping lemma guarantees some xyz split exists — the adversary chooses it. You must handle ALL valid splits, not just one convenient one.
Mistake 2 — Using pumping lemma to prove regularity: The lemma only proves non-regularity (by contradiction). Many non-regular languages also satisfy the pumping lemma conditions. To prove regularity, construct a DFA/NFA/RE.
Mistake 3 — i=0 vs i=2: Try i=0 (pump down) or i=2 (pump up) — whichever gives the contradiction for your language.

Classic languages proved non-regular by pumping lemma:

  • {aⁿbⁿ | n ≥ 0} — equal counts of a and b (shown above)
  • {aⁿ | n is prime} — prime-length strings
  • {ww | w ∈ {a,b}*} — strings that are their own repeat
  • {wwᴿ | w ∈ {a,b}*} — even-length palindromes
  • {aⁿ²| n ≥ 0} — perfect square length strings

Chapter 06
Context-Free Grammars — CFGs & Parse Trees

6.1   CFG — Formal Definition Core

CFG — 4-Tuple Definition

A CFG is G = (V, Σ, R, S) where:
V — finite set of variables (non-terminals), written in uppercase
Σ — terminal alphabet (disjoint from V), written in lowercase
R — finite set of production rules of the form A → α where A ∈ V and α ∈ (V∪Σ)*
S ∈ V — start variable

A CFG generates a language by starting with S and repeatedly replacing variables using production rules until only terminals remain. The language of the grammar L(G) is the set of all terminal strings derivable from S.

CFG for {aⁿbⁿ | n ≥ 0}:

G = ({S}, {a,b}, {S → aSb | ε}, S) Derivation of "aabb": S ⇒ aSb ⇒ aaSbb ⇒ aaεbb = aabb ✓ This is the canonical non-regular, context-free language. A DFA cannot recognize it; a CFG and PDA can.

CFG for balanced parentheses:

S → SS | (S) | ε Generates: ε, (), (()), ()(), ((())), (())(), ...

6.2   Derivations, Parse Trees & Ambiguity GATE Tested

Leftmost and Rightmost Derivations

Leftmost Derivation

Always replace the leftmost variable in the sentential form at each step.

Rightmost Derivation

Always replace the rightmost variable at each step. Rightmost derivations correspond to bottom-up parsing (LR parsers use this).

A parse tree is a tree representation of a derivation. The root is S, internal nodes are variables, leaves are terminals (or ε). The string generated = concatenation of leaves left to right.

Ambiguity

Ambiguous Grammar

A CFG G is ambiguous if there exists a string w ∈ L(G) that has more than one distinct parse tree (equivalently, more than one leftmost derivation, equivalently more than one rightmost derivation).

Ambiguous grammar for arithmetic: E → E+E | E*E | a

The string "a+a*a" has two parse trees:

Tree 1: (a+a)*a — multiply first E /|\ E * E | | a E+E → gives wrong precedence Tree 2: a+(a*a) — add first (correct math precedence) E /|\ E + E | | a E*E

This grammar is ambiguous. Fix by enforcing precedence: E → E+T | T, T → T*F | F, F → a.

Inherently Ambiguous Languages

Some context-free languages are inherently ambiguous — every CFG for them is ambiguous. No unambiguous grammar exists. Example: {aⁱbʲcᵏ | i=j or j=k}. GATE occasionally tests whether you can identify ambiguity vs inherent ambiguity.

6.3   Normal Forms — CNF & GNF GATE Tested

Chomsky Normal Form (CNF)

CNF

Every production has the form A → BC (two variables) or A → a (single terminal). No ε-productions except possibly S → ε. Every CFG can be converted to CNF.

Why CNF matters: CYK (Cocke-Younger-Kasami) algorithm — O(n³) parsing algorithm — requires the grammar in CNF. Also used in proofs about CFLs.

CNF Conversion Steps:

  1. 1
    Eliminate ε-productions (except S → ε if ε ∈ L(G)).
  2. 2
    Eliminate unit productions (A → B where B is a single variable).
  3. 3
    Eliminate useless symbols (unreachable or non-generating).
  4. 4
    Convert long productions: A → BCD becomes A → BX, X → CD.
  5. 5
    Replace terminals in mixed productions: A → aB becomes A → X_aB, X_a → a.

Greibach Normal Form (GNF)

GNF

Every production has the form A → aα where a is a terminal and α ∈ V*. Every production starts with a terminal. Every CFG can be converted to GNF. Used to construct PDAs from CFGs directly.

Chapter 07
Pushdown Automata — PDAs & CFLs

7.1   PDA — Formal Definition Core

PDA — 7-Tuple Definition

A PDA is M = (Q, Σ, Γ, δ, q₀, Z₀, F) where:
Q — finite set of states
Σ — input alphabet
Γ — stack alphabet
δ: Q × (Σ∪{ε}) × Γ → 2^(Q×Γ*) — transition function
q₀ — start state
Z₀ ∈ Γ — initial stack symbol (bottom of stack marker)
F ⊆ Q — accept states

A PDA is an NFA with a stack. At each step it reads (optionally) one input symbol, pops the top of the stack, and pushes a string of stack symbols (possibly empty = pop without push). This stack gives unbounded memory for counting — which is why PDAs can recognize {aⁿbⁿ} but DFAs cannot.

Acceptance by Final State

String accepted if PDA is in an accept state after reading all input. Stack contents don't matter at acceptance.

Acceptance by Empty Stack

String accepted if the stack is empty after reading all input. Accept states don't matter. Both definitions are equivalent in power.

7.2   PDA Design & Traces GATE Tested

PDA for L = {aⁿbⁿ | n ≥ 1} by empty stack:

States: q0 (reading a's), q1 (reading b's) Σ = {a,b}, Γ = {A, Z₀} Transitions (input, pop → push): δ(q0, a, Z₀) = {(q0, AZ₀)} ← first 'a': push A onto Z₀ δ(q0, a, A) = {(q0, AA)} ← subsequent a's: push A δ(q0, b, A) = {(q1, ε)} ← first 'b': pop A (switch mode) δ(q1, b, A) = {(q1, ε)} ← subsequent b's: pop A δ(q1, ε, Z₀) = {(q1, ε)} ← empty stack if no A left: accept Trace of "aabb": Config: (q0, aabb, Z₀) → read 'a': (q0, abb, AZ₀) → read 'a': (q0, bb, AAZ₀) → read 'b': (q1, b, AZ₀) → read 'b': (q1, ε, Z₀) → ε-move: (q1, ε, ε) ← ACCEPT (empty stack)

CFG → PDA Conversion

For every CFG G, there is an equivalent PDA M with L(M) = L(G). The construction uses a single-state PDA that simulates leftmost derivation:

  1. 1
    Push start symbol S onto stack.
  2. 2
    If top of stack is a variable A: nondeterministically choose a production A → α and replace A with α (push α in reverse).
  3. 3
    If top of stack is a terminal a: read 'a' from input and pop it. If they don't match, this path dies.
  4. 4
    Accept when stack is empty (if ε ∈ L) or just Z₀ remains.
Deterministic PDA vs Nondeterministic PDA

Unlike DFA and NFA (which are equivalent), DPDA ≠ NPDA in power. The language {ww^R | w ∈ {a,b}*} (even-length palindromes) requires an NPDA — no DPDA can recognize it. DPDA recognizes a strict subset of CFLs called deterministic CFLs. This is a key distinction: nondeterminism gives extra power for PDAs, unlike for finite automata.

CFL ∩ Regular = CFL — Extremely Useful

While CFLs are NOT closed under intersection with each other, they ARE closed under intersection with a regular language. This is used constantly to simplify CFL proofs. Example: to show {aⁿbⁿcⁿ} is not a CFL, intersect with the regular language a*b*c* and apply the CFL pumping lemma.

8.2   Pumping Lemma for CFLs GATE Tested

Pumping Lemma for CFLs — Statement

If L is a CFL, there exists a pumping length p such that for every w ∈ L with |w| ≥ p, there exists a decomposition w = uvxyz satisfying:
(1) |vxy| ≤ p    (2) |vy| ≥ 1 (v and y are not both empty)    (3) for all i ≥ 0, uvⁱxyⁱz ∈ L

The CFL pumping lemma has two strings to pump (v and y) instead of one. Both must be pumped together (same i). The middle part x between them doesn't get pumped.

Prove {aⁿbⁿcⁿ | n ≥ 0} is not a CFL.

Assume CFL. Let p be the pumping length. Choose w = aᵖbᵖcᵖ ∈ L (if it were). |w| = 3p ≥ p. w = uvxyz with |vxy| ≤ p, |vy| ≥ 1. Since |vxy| ≤ p and |w| = 3p, vxy spans at most two different character types (a,b or b,c — can't span all three of length p each). Case 1: v and y contain only a's and b's (no c's). Pump i=2: number of a's or b's increases, c's stay at p. → more a's or b's than c's → NOT in L. Contradiction. Case 2: v and y contain only b's and c's (no a's). Pump i=2: b's or c's increase, a's stay at p. → NOT in L. Contradiction. All cases lead to contradiction → {aⁿbⁿcⁿ} is NOT a CFL. ∎

Recursive vs Recursively Enumerable

Within Type 0: Recursive (decidable) languages have TMs that always halt (accept or reject). Recursively Enumerable languages have TMs that accept if the string is in L, but may loop forever if not. RL ⊂ CFL ⊂ CSL ⊂ Recursive ⊂ RE. These distinctions are covered fully in File 2 (Turing Machines & Decidability).

9.3   GATE Question Patterns

  • DFA design + min states: "Minimum states in DFA for strings containing '01' as substring" — count states in your mental DFA. Very frequent 1-mark NAT.
  • NFA→DFA subset construction: Given NFA, compute DFA states using subset construction. 2-mark question with full trace.
  • RE to language / language to RE: "Which RE describes strings with even number of a's?" Work it out: (b* a b* a b*)* · b*.
  • Pumping lemma proof: "Prove L is/is not regular using pumping lemma." Full proof expected — choose w, handle all splits, find i.
  • Closure property questions: "If L₁ is regular and L₂ is CFL, is L₁ ∩ L₂ always CFL?" — Yes (CFL closed under ∩ with regular).
  • Grammar classification: Given productions, identify grammar type (0/1/2/3) and corresponding automaton.
  • Ambiguity identification: Given grammar + string, find two distinct parse trees to prove ambiguity.
  • Language classification: Given a language description, determine highest Chomsky type and simplest recognizer.
