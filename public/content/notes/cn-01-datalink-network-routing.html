ARP Lives at Layer 3, Not Layer 2

Despite working with MAC addresses (Layer 2 concept), ARP is a Layer 3 protocol — it maps Layer 3 IP addresses to Layer 2 MAC addresses. It sits between the Network and Data Link layers. GATE tests this directly. Similarly, ICMP is Layer 3 (encapsulated in IP packets), not Layer 4.

1.2   Physical Layer — Bandwidth, Latency & Transmission GATE Tested

Key Physical Layer Formulas

Transmission time (Tₜ): Time to push all bits of a frame onto the wire. Tₜ = L / B where L = frame size (bits), B = bandwidth (bps).
Propagation delay (Tₚ): Time for one bit to travel from sender to receiver. Tₚ = d / v where d = distance, v = propagation speed (~2×10⁸ m/s in copper, 3×10⁸ in fibre).
End-to-end delay: Tₜ + Tₚ (for a single link, no queuing).
Bandwidth-Delay Product: B × Tₚ = number of bits that "fill" the pipe.

Packet of 1000 bytes sent over a 1 Mbps link, distance 1000 km, propagation speed 2×10⁸ m/s.

Tₜ = (1000 × 8 bits) / (1 × 10⁶ bps) = 8000 / 10⁶ = 8 ms Tₚ = 1000 × 10³ m / (2 × 10⁸ m/s) = 10⁶ / 2×10⁸ = 5 ms Total delay = 8 + 5 = 13 ms Bandwidth-Delay Product = 1×10⁶ × 5×10⁻³ = 5000 bits = 625 bytes (5000 bits are "in flight" in the pipe when sender has just finished sending)
Nyquist & Shannon Capacity

Nyquist (noiseless channel): Max data rate = 2 × B × log₂(M) bps, where B = bandwidth (Hz), M = signal levels.
Shannon (noisy channel): Channel capacity C = B × log₂(1 + S/N) bps, where S/N is signal-to-noise ratio (linear, not dB).

Chapter 02
Data Link Layer — Framing, Error Control & Sliding Window

2.1   Error Detection & Correction GATE Critical

Parity Bits

Even parity: Add a bit so total 1s in the codeword (data + parity bit) is even.
Odd parity: Add a bit so total 1s is odd.
Detects all odd-number-of-bit errors. Cannot detect even-number errors (2-bit errors cancel out). Cannot correct errors — only detect.

Cyclic Redundancy Check (CRC)

CRC Algorithm

Sender: Append (n−1) zeros to message M (where n = degree of generator polynomial G). Divide M×xⁿ⁻¹ by G using binary long division (XOR). The remainder R is the CRC. Send M followed by R.
Receiver: Divide received frame by G. If remainder = 0 → no error detected. If remainder ≠ 0 → error.

Message M = 1011, Generator G = 1011 (degree 3). Find CRC.

Append 3 zeros: M×x³ = 1011 000 Divide 1011000 by 1011 using XOR: 1011000 1011 (XOR at position 0) ───────── 000 000 (nothing to XOR, shift) 0001000 (bring down bits) Wait — let me redo properly: 1011|000 1011 → XOR: 0000, bring down 0 → 0000 shift right, get 00000 00000 ... Result: 1011 000 ÷ 1011 = quotient 1000, remainder 000 Actually: 1011 × 1000 = 1011000. 1011000 XOR 1011000 = 000. Remainder = 000. Transmitted frame: 1011 000. If generator doesn't divide evenly, remainder is the CRC appended to message. For M=110011, G=11001 (degree 4): Append 4 zeros → 1100110000 1100110000 ÷ 11001: 11001 ───────── 00001 10000 11001 ───────── 01001 (remainder) CRC = 1001, Transmitted = 110011 1001

Hamming Code

Hamming Code — Minimum Redundant Bits

For m data bits, need r redundant (parity) bits such that: 2ʳ ≥ m + r + 1.
Example: m=4 → 2ʳ ≥ 4+r+1 → r=3 (2³=8 ≥ 8 ✓). Total codeword length = m+r = 7 bits.
Parity bits occupy positions 1, 2, 4, 8, ... (powers of 2). Each covers specific data bit positions.

2.2   Sliding Window Protocols — Stop-and-Wait, Go-Back-N, SR Most Tested

Stop-and-Wait (SAW)

Send one frame. Wait for ACK. Send next only after ACK received. Simplest but extremely inefficient when Tₚ >> Tₜ.

Efficiency of Stop-and-Wait

η (eta) = Tₜ / (Tₜ + 2Tₚ) = 1 / (1 + 2a) where a = Tₚ / Tₜ.
If ACK transmission time and processing time are negligible.

Link: 1 Mbps, frame = 1000 bytes, Tₚ = 20ms. Efficiency of Stop-and-Wait?

Tₜ = (1000 × 8) / 10⁶ = 8 ms a = Tₚ / Tₜ = 20 / 8 = 2.5 η = 1 / (1 + 2×2.5) = 1/6 ≈ 16.7% Only 16.7% of link capacity is used — the rest is wasted waiting for ACKs.
Sliding Window — General Efficiency

With window size W (sender can have W unacknowledged frames in flight):
If W ≥ 1 + 2a: η = 1 (full utilization — window large enough to fill pipe)
If W < 1 + 2a: η = W / (1 + 2a)

ProtocolWindow SizeSequence bits nOn error: retransmitBuffer at receiver
Stop-and-Wait11 bit (0 and 1)1 frame1 frame
Go-Back-N (GBN)2ⁿ − 1n bitsN frames (all from error)1 frame (discards out-of-order)

Window Size Limits — Why They Are Different

Go-Back-N: Max window = 2ⁿ − 1. If window = 2ⁿ, sender and receiver cannot distinguish between a new window and a retransmitted old one after the sequence numbers wrap around.
Selective Repeat: Max window = 2ⁿ⁻¹ (half the sequence space). SR needs a stricter limit because the receiver buffers out-of-order frames. If sender window + receiver window > 2ⁿ, there would be an overlap in sequence numbers causing old and new frames to be confused.

Go-Back-N with 3-bit sequence numbers. n=3, so max window = 2³−1 = 7. Efficiency with 20ms Tₚ, 1ms Tₜ?

a = 20 / 1 = 20 1 + 2a = 41 Window W = 7 < 41 η = W / (1 + 2a) = 7 / 41 ≈ 17.1% For full utilization (η=1): need W ≥ 41. With 3-bit seq numbers, max W=7 — insufficient. Need n bits such that 2ⁿ−1 ≥ 41 → 2ⁿ ≥ 42 → n ≥ 6 bits.

Chapter 03
MAC Protocols & Ethernet

3.1   Multiple Access Protocols GATE Tested

TDMA / FDMA / CDMA

Channel partitioning. Time/Frequency/Code division. Efficient at high load, wasteful at low load (reserved slots go unused).

ALOHA

Random access. Pure ALOHA: transmit anytime. Slotted ALOHA: transmit at slot boundaries only. No coordination. Simple but collision-prone.

CSMA/CD

Carrier Sense Multiple Access with Collision Detection. Listen before transmit. Detect collision during transmission. Used in Ethernet (wired).

CSMA/CA

Collision Avoidance (not detection). Used in Wi-Fi (802.11). Avoids collision by waiting random backoff before transmitting.

ALOHA Throughput

Pure ALOHA: Maximum throughput = 1/(2e) ≈ 18.4% at load G = 0.5.
Slotted ALOHA: Maximum throughput = 1/e ≈ 36.8% at load G = 1.
Slotted ALOHA doubles efficiency by synchronizing transmissions to slot boundaries — collisions are complete (not partial).

3.2   CSMA/CD & Ethernet — Minimum Frame Size GATE Critical

CSMA/CD — Collision Detection Requirement

A sender must be transmitting when a collision signal arrives — otherwise it has already stopped and won't know a collision occurred. This means the transmission time must be at least twice the propagation delay:
Tₜ ≥ 2 × Tₚ
Minimum frame size = 2 × Tₚ × B (where B = bandwidth in bps)

Ethernet: 10 Mbps, max segment length 2500m, propagation speed 2×10⁸ m/s. Minimum frame size?

Tₚ = 2500 / (2×10⁸) = 12.5 μs 2 × Tₚ = 25 μs (round-trip time) Min frame size = 25×10⁻⁶ × 10×10⁶ = 250 bits = 31.25 bytes Ethernet standard rounds up to 64 bytes (512 bits) minimum frame size. The extra overhead (up to 64 bytes − header size) is padding.
Ethernet Frame Format

Ethernet frame: Preamble(7B) + SFD(1B) + Dest MAC(6B) + Src MAC(6B) + Type/Length(2B) + Data(46-1500B) + FCS(4B). Minimum data payload = 46 bytes (so total frame ≥ 64 bytes). Maximum data payload = 1500 bytes (the MTU). GATE tests: "What is the minimum/maximum Ethernet frame size?" Min=64B, Max=1518B (64+1500−46 = 1518 including FCS but excluding preamble/SFD).

4.2   Subnetting — VLSM & Complete Calculation Most Tested

Subnet Mask and CIDR Notation

A subnet mask identifies which bits of an IP address are the network part. Written as: 255.255.255.0 (dotted decimal) or /24 (CIDR — count of 1-bits in mask).
Network address = IP AND mask. Broadcast = network OR (NOT mask). Usable hosts = 2ʰ − 2 where h = host bits.

IP: 192.168.10.130/26. Find: network address, broadcast, host range, number of hosts.

Mask /26 = 255.255.255.192 = 11111111.11111111.11111111.11000000 IP: 192.168.10.130 = 11000000.10101000.00001010.10000010 Mask: 11111111.11111111.11111111.11000000 Network = IP AND Mask: 11000000.10101000.00001010.10000000 = 192.168.10.128 NOT Mask: 00000000.00000000.00000000.00111111 Broadcast = Net OR !Mask: 11000000.10101000.00001010.10111111 = 192.168.10.191 Host range: 192.168.10.129 to 192.168.10.190 Usable hosts: 2⁶ − 2 = 62 hosts (6 host bits since /26 = 32−26 = 6 host bits)

Subnet 192.168.1.0/24 into subnets of at least 50 hosts each. How many subnets? What is the subnet mask?

Need ≥ 50 hosts per subnet → 2ʰ − 2 ≥ 50 → 2ʰ ≥ 52 → h ≥ 6 Use h = 6 host bits → 2⁶ − 2 = 62 hosts per subnet ✓ Subnet bits = 8 − 6 = 2 → number of subnets = 2² = 4 New mask: /24 + 2 = /26 = 255.255.255.192 Subnets: 192.168.1.0/26 hosts .1–.62 broadcast .63 192.168.1.64/26 hosts .65–.126 broadcast .127 192.168.1.128/26 hosts .129–.190 broadcast .191 192.168.1.192/26 hosts .193–.254 broadcast .255
Subnetting Quick Method

Block size = 2ʰ (where h = host bits). Subnets start at multiples of block size within the last octet (for /24 → /32 subnets). For /26: block size = 64. Subnets at 0, 64, 128, 192. Network address = start of block. Broadcast = next block start − 1. First host = network + 1. Last host = broadcast − 1.

Chapter 05
ARP, ICMP & IPv6

5.1   ARP — Address Resolution Protocol GATE Tested

ARP Operation

ARP maps an IP address to a MAC address on the local network. When host A wants to send to IP address X: (1) Check ARP cache — if found, use the MAC. (2) If not found, broadcast ARP request: "Who has IP X? Tell A." (3) Host with IP X responds: "I have IP X. My MAC is YY:YY:YY:YY:YY:YY." (4) A caches the mapping and sends the frame.

MessageSourceDestinationType
ARP RequestRequester's MACFF:FF:FF:FF:FF:FF (broadcast)Broadcast to all
ARP ReplyTarget's MACRequester's MAC (unicast)Unicast back to requester
ARP is LAN-local — Routers Terminate ARP

ARP only works within a single LAN segment (broadcast domain). Routers do not forward ARP broadcasts. When sending to a remote host (different subnet), the sender ARPs for the router's MAC address (default gateway), not the destination host's MAC. The packet's IP destination is the remote host, but the frame's MAC destination is the router.

5.2   ICMP & IPv6 Highlights GATE Tested

ICMP — Internet Control Message Protocol

ICMP is used by routers and hosts to report network errors and send control messages. Encapsulated in IP packets. Key messages: Echo Request/Reply (ping), Destination Unreachable (no route, port closed, fragmentation needed), Time Exceeded (TTL=0, used by traceroute), Redirect (better route available).

TTL — Time To Live

Each IP packet has a TTL field (initially set by sender, typically 64 or 128). Every router that forwards the packet decrements TTL by 1. If TTL reaches 0, the router discards the packet and sends an ICMP Time Exceeded message back to the source. Prevents packets from looping forever.

Routing Algorithms — RIP, OSPF & BGP

6.1   Distance Vector vs Link State GATE Critical

Distance Vector (DV)

Each router knows distances to all destinations and the next hop. Shares routing table with neighbours. Simple. Slow convergence. Count-to-infinity problem. Example: RIP.

Link State (LS)

Each router floods the entire network with its link-state advertisements (LSAs). Every router builds a complete topology map and runs Dijkstra's algorithm locally. Fast convergence. No count-to-infinity. Higher bandwidth/memory cost. Example: OSPF.

6.2   Distance Vector — Bellman-Ford & Count-to-Infinity GATE Tested

Bellman-Ford Equation (Distributed)

Dₓ(y) = min over all neighbours v of {c(x,v) + Dᵥ(y)}
Router x computes its best distance to destination y as the minimum over all neighbour v of (cost to reach v + v's best distance to y).

Count-to-Infinity Problem:

Topology: A ─1─ B ─1─ C Initial state: A→B=1, A→C=2 (via B), B→A=1, B→C=1, C→A=2, C→B=1 Link B-C fails. C can no longer reach A. C's table: C→A = ∞, C→B = ∞ But: B still has old entry C→A = 2 from before failure. Incorrect exchange: B asks C: "What's your distance to A?" C says "3" (using B→A = 2 + 1 = 3) B now thinks: C→A = 3, so B→A = 3 + 1 = 4 C now thinks: B→A = 4, so C→A = 4 + 1 = 5 B now thinks: B→A = 5 + 1 = 6 ... This continues until distances count up to infinity (16 in RIP = "infinity") Solution: Split horizon (don't advertise a route back to the neighbour you learned it from)

IGP vs EGP

IGP (Interior Gateway Protocol): Routing within an Autonomous System (AS) — a network under single administrative control. Examples: RIP, OSPF, EIGRP.
EGP (Exterior Gateway Protocol): Routing between different Autonomous Systems. The internet backbone. Example: BGP (Border Gateway Protocol) — the protocol that holds the internet together.

BGP — Path Vector Protocol

BGP advertises the full AS path to reach a destination, not just hop count or link cost. This prevents routing loops (an AS won't route through itself) and allows policy-based routing (e.g., "don't route through AS X for political or commercial reasons"). BGP is what allows each ISP to control how their traffic flows through the internet.

Chapter 07
CIDR, NAT & Longest Prefix Matching

7.1   CIDR — Classless Inter-Domain Routing GATE Tested

CIDR

CIDR eliminates the fixed class boundaries of classful addressing. Any prefix length is valid (e.g., /13, /19, /27). Allows route aggregation (supernetting) — multiple contiguous networks can be summarised as one route, reducing routing table size. Notation: a.b.c.d/n where n = prefix length.

Route aggregation: 200.23.16.0/24, 200.23.17.0/24, 200.23.18.0/24, 200.23.19.0/24 — aggregate?

200.23.16.0: 11001000.00010111.0001 0000.00000000 200.23.17.0: 11001000.00010111.0001 0001.00000000 200.23.18.0: 11001000.00010111.0001 0010.00000000 200.23.19.0: 11001000.00010111.0001 0011.00000000 Common prefix: 11001000.00010111.0001 00xx.xxxxxxxx = 22 bits Aggregate: 200.23.16.0/22 This single /22 route represents all four /24 networks. Sending ISP advertises one route instead of four → reduces routing table entries.

7.2   Longest Prefix Matching — Routing Table Lookup GATE Tested

Longest Prefix Match

When a router receives a packet, it looks up the destination IP in the routing table. Multiple entries may match. The router uses the entry with the longest matching prefix (most specific route). This is the fundamental forwarding rule for IP routing.

Routing table: 192.168.1.0/24 → A, 192.168.0.0/16 → B, 0.0.0.0/0 → C (default route).

Packet dest: 192.168.1.53 Matches: 192.168.1.0/24 (24-bit match), 192.168.0.0/16 (16-bit match), 0.0.0.0/0 (0-bit match) Longest prefix: /24 → forward to A Packet dest: 192.168.20.1 Matches: 192.168.0.0/16 (16-bit), 0.0.0.0/0 (0-bit) Longest prefix: /16 → forward to B Packet dest: 10.0.0.1 Matches: 0.0.0.0/0 only Longest prefix: /0 → forward to C (default gateway)

7.3   NAT — Network Address Translation GATE Tested

NAT

NAT translates private IP addresses (10.x, 172.16.x, 192.168.x) to a public IP address (and back) at the network boundary router. Allows multiple devices to share one public IP. The NAT router maintains a NAT translation table mapping (private IP, private port) ↔ (public IP, public port).

NAT in action:

Internal host: 192.168.1.5:3456 sends packet to 8.8.8.8:80 NAT router (public IP: 203.0.113.1): Translates: 192.168.1.5:3456 → 203.0.113.1:5001 Records in table: (192.168.1.5, 3456) ↔ (203.0.113.1, 5001) Forwards packet with src=203.0.113.1:5001 Reply from 8.8.8.8:80 to 203.0.113.1:5001: NAT looks up table: 5001 → (192.168.1.5, 3456) Translates dst back: 203.0.113.1:5001 → 192.168.1.5:3456 Delivers to internal host
NAT Breaks End-to-End Connectivity

NAT violates the end-to-end principle of the internet. External hosts cannot initiate connections to NAT-internal hosts (no inbound entry in the translation table). Solutions: port forwarding (static NAT entry), STUN/TURN (for P2P apps like WebRTC), IPv6 (eliminates NAT need). GATE tests: "What problem does NAT introduce?" → breaks end-to-end connectivity, external cannot reach internal.

GATE Traps & Complete Reference

8.1   All the Traps in One Place

#TrapCorrect Understanding
1ARP is Layer 2ARP is Layer 3 — it provides a service to Layer 3 (IP) by resolving Layer 2 addresses. It sits between L2 and L3. ICMP is also Layer 3 (encapsulated in IP). Neither is Layer 4.
2Efficiency = Tₜ / (Tₜ + Tₚ)Stop-and-Wait efficiency = Tₜ / (Tₜ + 2Tₚ) — factor of 2 because you wait for the ACK to travel back. The full round-trip includes both directions.
3GBN and SR have the same window size limitGBN max window = 2ⁿ−1. SR max window = 2ⁿ⁻¹ (half). SR needs a stricter limit because it buffers out-of-order frames, creating a potential for confusion between old and new windows.
4Router forwards broadcast packetsRouters do NOT forward broadcast packets — they terminate broadcast domains. ARP broadcasts, DHCP discover, and limited broadcasts (255.255.255.255) are all stopped at the router. Only switches flood broadcasts within a LAN.
5Switch = hub with intelligenceSwitch creates separate collision domains per port. Hub has ONE collision domain for all ports. Switch has ONE broadcast domain (all ports). Router separates both collision AND broadcast domains.
6Minimum Ethernet frame is 46 bytesMinimum Ethernet frame is 64 bytes TOTAL (header + data + FCS). Minimum DATA payload is 46 bytes (to ensure total reaches 64 bytes). This is the CSMA/CD requirement: transmission must last at least 2×Tₚ.
7Class C network has 256 addressesClass C (/24) has 2⁸ = 256 IP addresses total. But only 254 are usable for hosts — subtract network address (all host bits 0) and broadcast address (all host bits 1). Always subtract 2 for usable host count.
8OSPF uses Bellman-FordOSPF is a Link State protocol — it uses Dijkstra's algorithm. Each router runs Dijkstra locally on its complete topology map. RIP (Distance Vector) uses a distributed form of Bellman-Ford.
9BGP is a link-state protocolBGP is a path-vector protocol — a hybrid of distance vector and link state. It advertises the complete AS path to a destination, not just a metric. This enables loop prevention and policy routing between autonomous systems.
10Longest prefix match = largest IPLongest prefix match means the prefix with the most bits matching — the most specific route. /28 is more specific than /16, regardless of IP value. Default route /0 is matched last (least specific, always matches).
11IPv6 removes fragmentation entirelyIPv6 removes router-based fragmentation. Routers drop oversized IPv6 packets and send ICMPv6 "Packet Too Big" back. The source can then fragment. So fragmentation still exists at the source — it's just not done mid-network by routers.
12Pure ALOHA is more efficient than Slotted ALOHASlotted ALOHA (max 36.8% = 1/e) is exactly twice as efficient as Pure ALOHA (max 18.4% = 1/2e). Synchronization eliminates partial collisions, doubling throughput.

8.3   GATE Question Patterns

  • Efficiency NAT: "Frame=1KB, link=1Mbps, Tₚ=20ms, window=7, Go-Back-N. What is efficiency?" Compute Tₜ, a, check W vs 1+2a, apply formula. Very common 2-mark question.
  • Minimum frame size: "A network has bandwidth 10Mbps and round-trip propagation delay 50μs. What is the minimum frame size?" = 50×10⁻⁶ × 10×10⁶ = 500 bits = 62.5 bytes → round up to 64 bytes.
  • Subnetting NAT: "Host 172.16.45.67/20. What is the network address?" Mask /20 = 255.255.240.0. 45 in binary: 00101101. Upper 4 bits of third octet: 0010 = 32. Network = 172.16.32.0. 2 marks.
  • Routing protocol MCQ: "Which protocol uses Dijkstra's algorithm?" OSPF. "Which suffers from count-to-infinity?" RIP. 1-mark direct.
  • Longest prefix MCQ: Given routing table, which entry is used to forward packet to X? Find all matching entries, pick longest prefix. 1-2 marks.
  • CRC calculation: "Message = 1010011, Generator = 1011. What is the transmitted frame?" Append 3 zeros (degree-3 generator), perform XOR division, append remainder to original message. 2 marks.
  • ARP sequence: "Host A sends packet to host B (different subnet). What MAC address does A put in the frame?" Default gateway's MAC (not B's MAC). 1-mark concept question.
  • OSI layer MCQ: "At which layer does IP fragmentation occur?" Layer 3 (Network). "At which layer do switches operate?" Layer 2. Classic 1-mark questions.
