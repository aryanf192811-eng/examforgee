Chapter 01
First-Order ODEs — Separation, Linearity & Exactness

1.1   ODE Classification & Order Core

ODE Classification

An ODE (Ordinary Differential Equation) involves a function of ONE variable and its derivatives.
Order: Order of the highest derivative. y'' + 2y' + y = 0 is 2nd order.
Degree: Power of the highest-order derivative (after clearing fractions/radicals). (y'')² + y' = x is 2nd order, degree 2.
Linear: The dependent variable y and all its derivatives appear with exponent 1 only, with no products between them. y'' + xy' + y = sin x is linear. y·y' = x is nonlinear.

1.2   Methods for First-Order ODEs GATE Tested

1. Separable Equations

Form and Method

dy/dx = f(x)·g(y). Separate variables: dy/g(y) = f(x)dx. Integrate both sides.
Works when the RHS factors into a product of a function of x alone and a function of y alone.

Solve dy/dx = xy, y(0) = 2:

Separate: dy/y = x dx Integrate: ln|y| = x²/2 + C₁ Solve: y = Ae^(x²/2) where A = e^C₁ Apply IC: 2 = Ae⁰ = A → A = 2 Solution: y = 2e^(x²/2)

2. Linear First-Order ODE

Form: dy/dx + P(x)y = Q(x)

Multiply both sides by the integrating factor μ(x) = e^(∫P(x)dx).
The LHS becomes d/dx[μ(x)·y]. Integrate both sides:
y = (1/μ(x)) ∫ μ(x)·Q(x)dx + C/μ(x)

Solve dy/dx + 2y = 4x:

P(x) = 2, Q(x) = 4x μ(x) = e^(∫2dx) = e^(2x) Multiply: e^(2x)(dy/dx + 2y) = 4x·e^(2x) LHS = d/dx[y·e^(2x)] Integrate: y·e^(2x) = ∫4x·e^(2x)dx ∫4x·e^(2x)dx (by parts, u=4x, dv=e^(2x)dx): = 4x·(e^(2x)/2) − ∫4·(e^(2x)/2)dx = 2xe^(2x) − e^(2x) + C So: y·e^(2x) = 2xe^(2x) − e^(2x) + C y = 2x − 1 + Ce^(−2x)

3. Bernoulli Equation

Form: dy/dx + P(x)y = Q(x)·yⁿ

Substitute v = y^(1−n). Then dv/dx = (1−n)y^(−n) dy/dx. The equation transforms to a linear ODE in v: dv/dx + (1−n)P(x)v = (1−n)Q(x).

4. Exact Equations

Form: M(x,y)dx + N(x,y)dy = 0

Exact iff ∂M/∂y = ∂N/∂x. If exact, there exists F(x,y) such that ∂F/∂x = M and ∂F/∂y = N. Solution: F(x,y) = C.
Find F: integrate M w.r.t. x (treating y as constant), then use ∂F/∂y = N to find the y-dependent terms.

Check if (2xy+y²)dx + (x²+2xy)dy = 0 is exact and solve:

M = 2xy+y², N = x²+2xy ∂M/∂y = 2x + 2y, ∂N/∂x = 2x + 2y → EXACT ✓ Find F: ∂F/∂x = M = 2xy+y² Integrate w.r.t. x: F = x²y + xy² + g(y) ∂F/∂y = x² + 2xy + g'(y) = N = x² + 2xy → g'(y) = 0 → g(y) = constant Solution: x²y + xy² = C
Integrating Factor for Non-Exact Equations

If ∂M/∂y ≠ ∂N/∂x, multiply by an integrating factor μ to make it exact.
If (∂M/∂y − ∂N/∂x)/N = f(x) depends only on x: μ = e^(∫f(x)dx).
If (∂N/∂x − ∂M/∂y)/M = g(y) depends only on y: μ = e^(∫g(y)dy).

Chapter 02
Second-Order ODEs — Homogeneous & Non-Homogeneous

2.1   Linear Second-Order ODEs with Constant Coefficients GATE Critical

Standard Form

ay'' + by' + cy = f(x)
Homogeneous: f(x) = 0. General solution = complementary function yc.
Non-homogeneous: f(x) ≠ 0. General solution y = yc + yp (complementary + particular).

Step 1 — Complementary Function (Characteristic Equation)

Characteristic Equation: aλ² + bλ + c = 0

Case 1: Two distinct real roots λ₁ ≠ λ₂:
yc = C₁e^(λ₁x) + C₂e^(λ₂x)

Case 2: Repeated real root λ₁ = λ₂ = λ:
yc = (C₁ + C₂x)e^(λx)

Case 3: Complex roots λ = α ± βi:
yc = e^(αx)(C₁cos(βx) + C₂sin(βx))

Solve y'' − 5y' + 6y = 0:

Characteristic: λ² − 5λ + 6 = 0 → (λ−2)(λ−3) = 0 Roots: λ₁=2, λ₂=3 (distinct real) yc = C₁e^(2x) + C₂e^(3x)

Solve y'' + 4y' + 4y = 0:

Characteristic: λ² + 4λ + 4 = 0 → (λ+2)² = 0 Repeated root: λ = −2 yc = (C₁ + C₂x)e^(−2x)

Solve y'' + 2y' + 5y = 0:

Characteristic: λ² + 2λ + 5 = 0 λ = (−2 ± √(4−20))/2 = (−2 ± √(−16))/2 = −1 ± 2i Complex: α=−1, β=2 yc = e^(−x)(C₁cos(2x) + C₂sin(2x))

Step 2 — Particular Solution (Method of Undetermined Coefficients)

Chapter 03
Root Finding — Bisection & Newton-Raphson

3.1   Bisection Method GATE Tested

Bisection Method

Find a root of f(x) = 0 in [a,b] where f(a)·f(b) < 0 (opposite signs guarantee a root by IVT).
At each step: c = (a+b)/2.
If f(c) = 0: c is the root.
If f(a)·f(c) < 0: root is in [a,c], set b=c.
Else: root is in [c,b], set a=c.
Error after n steps: |error| ≤ (b−a)/2ⁿ
Steps for ε accuracy: n ≥ log₂((b−a)/ε)

Find root of f(x) = x³ − x − 2 in [1,2] with 3 iterations:

f(1) = 1−1−2 = −2 < 0 f(2) = 8−2−2 = 4 > 0 → root in [1,2] ✓ Iter 1: c = (1+2)/2 = 1.5. f(1.5) = 3.375−1.5−2 = −0.125 < 0 Root in [1.5, 2] Iter 2: c = (1.5+2)/2 = 1.75. f(1.75) = 5.359−1.75−2 = 1.609 > 0 Root in [1.5, 1.75] Iter 3: c = (1.5+1.75)/2 = 1.625. f(1.625) = 4.291−1.625−2 = 0.666 > 0 Root in [1.5, 1.625] After 3 iterations: root ≈ 1.5625 (midpoint). Error ≤ (2−1)/2³ = 0.125
Bisection: Linear Convergence — 1 Bit Per Iteration

Bisection halves the interval each iteration — the number of correct binary digits increases by exactly 1 per iteration. This is called linear convergence. It is guaranteed to converge but slow. Newton-Raphson has quadratic convergence (roughly doubles the correct digits each iteration) but can diverge. Bisection is the safe fallback.

3.2   Newton-Raphson Method GATE Critical

Newton-Raphson Formula

Given initial guess x₀, iterate:
xₙ₊₁ = xₙ − f(xₙ)/f'(xₙ)
Geometric interpretation: draw the tangent line at (xₙ, f(xₙ)), find where it crosses the x-axis — that's xₙ₊₁.
Convergence: quadratic — errors roughly square each iteration. Fails if f'(xₙ) = 0 (horizontal tangent at iterate).

Find √2 using Newton-Raphson with x₀ = 1:

Find root of f(x) = x² − 2. f'(x) = 2x. Formula: xₙ₊₁ = xₙ − (xₙ² − 2)/(2xₙ) = (xₙ + 2/xₙ)/2 x₀ = 1 x₁ = (1 + 2/1)/2 = 1.5 x₂ = (1.5 + 2/1.5)/2 = (1.5 + 1.333)/2 = 1.4167 x₃ = (1.4167 + 2/1.4167)/2 = (1.4167 + 1.4118)/2 = 1.41421... Converges to √2 = 1.41421... in 3 steps from x₀=1. ✓ (Compare: bisection on [1,2] needs 10+ iterations for same accuracy)

Chapter 04
Numerical Integration — Trapezoidal & Simpson's Rules

4.1   Trapezoidal & Simpson's Rules GATE Tested

Approximate ∫_a^b f(x)dx when f has no closed-form antiderivative. Divide [a,b] into n equal subintervals of width h = (b−a)/n.

Trapezoidal Rule

Approximate f(x) by piecewise linear functions (trapezoids):
∫_a^b f(x)dx ≈ (h/2)[f(x₀) + 2f(x₁) + 2f(x₂) + ... + 2f(xₙ₋₁) + f(xₙ)]
Error: O(h²) — error proportional to h². Halving h reduces error by factor 4.
Exact for: Linear functions (degree ≤ 1).

Simpson's 1/3 Rule (n must be even)

Approximate f(x) by piecewise quadratic functions (parabolas) over pairs of subintervals:
∫_a^b f(x)dx ≈ (h/3)[f(x₀) + 4f(x₁) + 2f(x₂) + 4f(x₃) + 2f(x₄) + ... + 4f(xₙ₋₁) + f(xₙ)]
Pattern: 1, 4, 2, 4, 2, ..., 4, 1 (start/end are 1, alternating 4,2 in between)
Error: O(h⁴) — much more accurate than trapezoidal.
Exact for: Polynomials of degree ≤ 3 (cubic and below).

Estimate ∫_0^1 eˣ dx with n=4 using both rules:

Exact value: [eˣ]_0^1 = e − 1 ≈ 1.71828 h = (1−0)/4 = 0.25 x₀=0, x₁=0.25, x₂=0.5, x₃=0.75, x₄=1 f(x) = eˣ: f(x₀)=1.0000, f(x₁)=1.2840, f(x₂)=1.6487, f(x₃)=2.1170, f(x₄)=2.7183 Trapezoidal: (0.25/2)[1.0000 + 2(1.2840+1.6487+2.1170) + 2.7183] = 0.125[1.0000 + 2(5.0497) + 2.7183] = 0.125[1.0000 + 10.0994 + 2.7183] = 0.125 × 13.8177 = 1.7272 Error = |1.7272 − 1.7183| = 0.0089 Simpson's 1/3: (0.25/3)[1.0000 + 4(1.2840) + 2(1.6487) + 4(2.1170) + 2.7183] = (0.0833)[1.0000 + 5.1360 + 3.2974 + 8.4680 + 2.7183] = 0.0833 × 20.6197 = 1.7183 Error ≈ 0 (Simpson's is essentially exact here!)
Simpson's 3/8 Rule (n must be multiple of 3)

∫_a^b f(x)dx ≈ (3h/8)[f(x₀) + 3f(x₁) + 3f(x₂) + 2f(x₃) + 3f(x₄) + ... + f(xₙ)]
Pattern: 1, 3, 3, 2, 3, 3, 2, ..., 3, 3, 1. Exact for polynomials ≤ degree 3.

Chapter 05
Numerical ODEs — Euler & Runge-Kutta

5.1   Euler's Method & Runge-Kutta GATE Tested

Solve dy/dx = f(x,y), y(x₀) = y₀ numerically. Use step size h.

Euler's Method (First-Order)

yₙ₊₁ = yₙ + h·f(xₙ, yₙ)
Use the slope at the current point to step forward. Simple but low accuracy.
Error per step: O(h²). Global error: O(h) — first-order method.

Solve dy/dx = x + y, y(0)=1, step h=0.1, find y(0.2):

f(x,y) = x + y Step 1: x₀=0, y₀=1 y₁ = y₀ + h·f(x₀,y₀) = 1 + 0.1·(0+1) = 1 + 0.1 = 1.1 Step 2: x₁=0.1, y₁=1.1 y₂ = y₁ + h·f(x₁,y₁) = 1.1 + 0.1·(0.1+1.1) = 1.1 + 0.12 = 1.22 y(0.2) ≈ 1.22 Exact: y = 2eˣ − x − 1. y(0.2) = 2e^0.2 − 0.2 − 1 = 2(1.2214)−1.2 = 1.2428 Error = |1.2428 − 1.22| = 0.0228 (significant error for h=0.1)
Runge-Kutta Method (RK4 — Fourth Order)

The most widely used numerical ODE method. Uses four slope estimates (k₁,k₂,k₃,k₄) to compute a weighted average slope:
k₁ = h·f(xₙ, yₙ)
k₂ = h·f(xₙ + h/2, yₙ + k₁/2)
k₃ = h·f(xₙ + h/2, yₙ + k₂/2)
k₄ = h·f(xₙ + h, yₙ + k₃)
yₙ₊₁ = yₙ + (k₁ + 2k₂ + 2k₃ + k₄)/6
Global error: O(h⁴) — far superior to Euler's O(h).

One step of RK4: dy/dx = x+y, y(0)=1, h=0.2, find y(0.2):

f(x,y) = x+y, x₀=0, y₀=1, h=0.2 k₁ = h·f(0, 1) = 0.2·(0+1) = 0.2 k₂ = h·f(0.1, 1+0.1) = 0.2·(0.1+1.1) = 0.2·1.2 = 0.24 k₃ = h·f(0.1, 1+0.12) = 0.2·(0.1+1.12) = 0.2·1.22 = 0.244 k₄ = h·f(0.2, 1+0.244) = 0.2·(0.2+1.244) = 0.2·1.444 = 0.2888 y₁ = 1 + (0.2 + 2(0.24) + 2(0.244) + 0.2888)/6 = 1 + (0.2 + 0.48 + 0.488 + 0.2888)/6 = 1 + 1.4568/6 = 1 + 0.2428 = 1.2428 Exact: y(0.2) = 1.2428 ✓ (RK4 matches to 4 decimal places in ONE step)

Chapter 06
Numerical Linear Algebra — Gaussian Elimination & Iterative Methods

6.1   Gaussian Elimination & LU Decomposition GATE Tested

Gaussian Elimination

Transform the augmented matrix [A|b] to row-echelon form using elementary row operations, then back-substitute to find the solution. Three elementary row operations: (1) Swap two rows. (2) Multiply a row by a nonzero scalar. (3) Add a multiple of one row to another.

Solve: 2x+y−z=8, −3x−y+2z=−11, −2x+y+2z=−3:

Augmented matrix: [ 2 1 −1 | 8] [−3 −1 2 |−11] [−2 1 2 | −3] R2 ← R2 + (3/2)R1: [ 2 1 −1 | 8] [ 0 1/2 1/2| 1] [−2 1 2 | −3] R3 ← R3 + R1: [ 2 1 −1 | 8] [ 0 1/2 1/2| 1] [ 0 2 1 | 5] R3 ← R3 − 4·R2: [ 2 1 −1 | 8] [ 0 1/2 1/2| 1] [ 0 0 −1 | 1] Back-substitute: z=−1, y/2+z/2=1→y=3, 2x+3+1=8→x=2 Solution: x=2, y=3, z=−1 ✓
LU Decomposition

Factor A = LU where L is lower triangular (1s on diagonal) and U is upper triangular. Useful when solving Ax=b for multiple right-hand sides b — compute LU once, then solve Ly=b (forward substitution) and Ux=y (back substitution) for each b. Time: O(n³) for factorization, O(n²) per solve.

Iterative Methods — Gauss-Jacobi & Gauss-Seidel

Gauss-Jacobi Method

For Ax=b with A diagonally dominant: update ALL variables using values from the PREVIOUS iteration simultaneously.
xᵢ^(k+1) = (1/aᵢᵢ) · [bᵢ − Σⱼ≠ᵢ aᵢⱼ·xⱼ^(k)]

Gauss-Seidel Method

Same as Jacobi but use the MOST RECENTLY COMPUTED values immediately (within the same iteration). Faster convergence than Jacobi (roughly 2× fewer iterations). Convergence guaranteed if A is strictly diagonally dominant.

Diagonal Dominance — Convergence Condition

A matrix A is strictly diagonally dominant if |aᵢᵢ| > Σⱼ≠ᵢ |aᵢⱼ| for every row i. Both Jacobi and Gauss-Seidel are guaranteed to converge for strictly diagonally dominant matrices. Gauss-Seidel typically converges faster and is preferred. If the matrix is not diagonally dominant, reorder the equations to make it so if possible.

Mathematical Logic — Propositional & Predicate

8.1   Logic Quick Reference for EM GATE Tested

The full treatment of propositional logic, predicate logic, quantifiers, normal forms, functionally complete sets, and proof techniques is in Discrete Mathematics File 1. This section provides the EM-specific focus: formal systems, completeness, soundness, and decidability as they appear in Engineering Mathematics context.

Soundness and Completeness

A proof system is sound if every provable statement is true (no false theorems). A proof system is complete if every true statement is provable (Gödel completeness for first-order logic — if a formula is true in all models, it is provable).
Gödel's Incompleteness Theorem (1931): Any consistent formal system strong enough to express arithmetic contains true statements that cannot be proved within the system. Completeness and consistency cannot both hold simultaneously for sufficiently expressive systems.

Resolution — Automated Theorem Proving

Resolution is a complete inference method for propositional and first-order logic. From clauses (A ∨ B) and (¬A ∨ C), resolve to get (B ∨ C). To prove φ: negate φ, add it to the clause set, and apply resolution. If you derive the empty clause (contradiction), φ is proved. Resolution is the basis of logic programming (Prolog) and SAT solvers.

9.1   All the Traps in One Place

#TrapCorrect Understanding
1Order of ODE = highest power of derivativeOrder = highest ORDER (which derivative) present. Degree = power of that highest derivative (after clearing radicals). y'' + (y')³ = 0 is ORDER 2, DEGREE 1. (y'')² + y' = 0 is ORDER 2, DEGREE 2.
2If char polynomial root matches f(x), use the same trial solutionIf f(x) = Aeᵃˣ and a is a root of the characteristic equation (multiplicity m), the trial particular solution must be xᵐ·Aeᵃˣ — multiply by xᵐ to avoid collision with the complementary function. Forgetting this gives yp=0 after substitution.
3Bisection needs only f(a)·f(b) < 0Bisection also needs f to be CONTINUOUS on [a,b]. Without continuity, sign change doesn't guarantee a root (e.g., f(x)=1/x on [−1,1] has a sign change but no root — it has a discontinuity).
4Newton-Raphson always convergesNewton-Raphson can DIVERGE if: initial guess is far from root, f'(xₙ)≈0 (nearly horizontal tangent), the function has inflection points near the root, or the root is a repeated root (convergence slows to linear). Always verify convergence conditions.
5Simpson's rule works for any nSimpson's 1/3 rule requires n to be EVEN. Simpson's 3/8 rule requires n to be a multiple of 3. If n doesn't satisfy the requirement, you cannot directly apply that rule (use Trapezoidal or a different composite formula).
6Trapezoidal rule is more accurate than Simpson'sSimpson's 1/3 rule is MORE accurate: error O(h⁴) vs Trapezoidal's O(h²). Halving h reduces Trapezoidal error by 4× but Simpson's error by 16×. Simpson's is exact for cubics (degree ≤ 3); Trapezoidal is only exact for linears (degree ≤ 1).
7Euler's method error is O(h)Euler's LOCAL (per-step) truncation error is O(h²). The GLOBAL error (accumulated over all steps to reach a fixed point) is O(h) — one order lower due to accumulation. When comparing methods, use global error order. RK4 has local error O(h⁵) and global error O(h⁴).
8Gauss-Seidel and Jacobi converge under the same conditionsBoth converge for strictly diagonally dominant matrices. However, Gauss-Seidel converges roughly TWICE as fast as Jacobi (fewer iterations for same accuracy). For some matrices, one may converge while the other diverges — they are not equivalent. Gauss-Seidel is preferred in practice.
9Exact equations have ∂M/∂x = ∂N/∂yThe exactness condition is ∂M/∂y = ∂N/∂x (both with respect to the OTHER variable). M is the coefficient of dx, N is the coefficient of dy. ∂M/∂y = ∂N/∂x, NOT ∂M/∂x = ∂N/∂y. Mixing up which to differentiate with respect to is the most common exact equation error.
10Separable means RHS = f(x)/g(y)Separable means dy/dx = f(x)·g(y) — a PRODUCT (not quotient) of a function of x alone and a function of y alone. dy/dx = (x+y) is NOT separable (can't factor into f(x)·g(y)). dy/dx = x·y IS separable.
11Simpson's coefficient pattern: 1,2,4,2,4,...,1Simpson's 1/3 coefficient pattern is 1,4,2,4,2,...,4,1 (alternating 4 and 2 between endpoints). Many students write it as 1,2,4,2,4,...,1 (wrong order). Remember: endpoints are 1, then ALWAYS start with 4, alternate 2,4,2,4 until the last which is also 4, then endpoint 1.
12Gödel's Completeness ≡ Incompleteness theoremGödel proved TWO different theorems. COMPLETENESS (1929): First-order logic is semantically complete — every valid formula is provable. INCOMPLETENESS (1931): Sufficiently powerful consistent formal systems contain true but unprovable statements. These address different systems and different questions.

9.3   GATE Question Patterns

  • ODE classification MCQ: "What is the order and degree of (y'')² + 3y' = x?" Order=2, Degree=2. "Is dy/dx = x²+y² linear?" No (y² makes it nonlinear). 1-mark each.
  • Separable ODE NAT: "Solve dy/dx = x/y, y(0)=2. Find y(1)." Separate: y dy = x dx → y²/2 = x²/2 + C → y² = x²+C. IC: 4=0+C → C=4. y(1) = √5. 2 marks.
  • Char equation MCQ: "General solution of y''−4y=0?" Char: λ²−4=0 → λ=±2. y = C₁e^(2x)+C₂e^(−2x). 1 mark.
  • Bisection iterations NAT: "How many bisection iterations to achieve accuracy 10⁻³ on [0,2]?" n ≥ log₂(2/0.001) = log₂(2000) ≈ 10.97 → n=11. 2 marks.
  • Newton-Raphson iteration NAT: "f(x)=x²−3. x₀=2. Find x₁." x₁ = 2 − (4−3)/4 = 2−0.25 = 1.75. 1 mark.
  • Trapezoidal NAT: "Estimate ∫_0^2 x² dx using Trapezoidal with n=4." h=0.5, x: 0,0.5,1,1.5,2. f: 0,0.25,1,2.25,4. T=(0.5/2)[0+2(0.25+1+2.25)+4]=0.25[0+7+4]=2.75. Exact=8/3≈2.667. 2 marks.
  • Simpson's NAT: Same integral, Simpson's: (0.5/3)[0+4(0.25)+2(1)+4(2.25)+4]=0.1667[0+1+2+9+4]=2.667. Exact! 2 marks.
  • Euler's method NAT: "dy/dx=y, y(0)=1, h=0.1. Find y(0.3) using Euler." y₁=1+0.1(1)=1.1, y₂=1.1+0.1(1.1)=1.21, y₃=1.21+0.1(1.21)=1.331. 2 marks.
  • Logic MCQ: "Which of these is a tautology?" p∨¬p (always true). "Which is a contradiction?" p∧¬p (always false). 1 mark each.
  • Walks in graph NAT: "How many walks of length 2 from vertex 1 to vertex 3 in this graph?" Compute (A²)₁₃. 2 marks.
