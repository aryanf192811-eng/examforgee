Stage 0 — You start here
Complete Beginner — Never written C++ before
Open File 08 first, then File 01
You need two things simultaneously: a feel for what problems look like, and the C++ syntax to write solutions. File 08 gives you ~80 beginner problems with step-by-step thinking. File 01 gives you the language. Run them in parallel.
1
Stage 1 — Foundation
Know syntax, solving simple problems
Active files: File 01 (language) + File 08 (practice)
Variables, loops, functions, arrays, pointers. You can write programs but often don't know where to start on a new problem. Complete File 01 fully. Finish File 08's first 6 chapters. Understand every line you write.
2
Stage 2 — Logic Builder
Syntax is comfortable, logic is the bottleneck
Active files: File 07 (logic thinking) + File 08 Ch7–8 (harder problems)
This is where most people are stuck longest. You know the language but stare at blank screens. File 07 fixes this — it teaches you HOW to think about any problem before touching the keyboard. The 5-step framework, constraint reading, and pattern recognition are the most valuable 50 pages in this series for someone at this stage.
3
Stage 3 — OOP & Modern C++
Procedural programmer becoming a C++ programmer
Active files: File 02 (OOP) → File 03 (Advanced C++)
Classes, inheritance, polymorphism, templates, STL, smart pointers, lambdas, modern C++11–20 features. Read File 02 completely before File 03. These two files make you write production-quality C++ instead of C with classes.
4
Stage 4 — DSA
Comfortable with C++, learning algorithms
Active file: File 05 (DSA with C++)
This is where competitive programming starts. Linked lists, stacks, queues, trees, graphs, sorting, binary search, dynamic programming. Every algorithm is implemented from scratch in C++. Do not skip to this stage — you need Files 01–03 first or the code won't make sense.
5
Stage 5 — Expert C++
Solving DSA problems, want deeper C++ mastery
Active file: File 04 (Pure Mastery)
Undefined behavior, memory model, template metaprogramming, CRTP, performance engineering, ABI, design patterns. This is what separates good engineers from great ones. Read this file after File 03, not before. It builds on everything.
6
Stage 6 — Competitive Programming
ICPC / Codeforces level
Active file: File 06 (The Black Box)
Copy-paste contest templates. Binary lifting, centroid decomp, Dinic's max flow, Manacher, suffix arrays, NTT, convex hull trick, digit DP, game theory, ICPC strategy. This file assumes you've solved 300+ problems. It's a weapon, not a tutorial.

The one rule that matters most

Never move to the next file until you can explain every concept in the current one out loud — without looking at notes. Rushing to File 06 without File 05 mastery is the single most common mistake. The files are sequenced to build on each other. The sequence is the curriculum.

How to use Files 07 and 08 throughout your journey

Files 07 and 08 are not one-time reads — they are permanent references. File 07's signal→algorithm table and 20 rules should be open every time you sit down to practice. File 08's intermediate chapter should be revisited every few weeks as you get stronger — problems that seemed hard will feel obvious, which tells you exactly how much you've grown.
