Graph Fundamentals & Terminology

1.1   Graph — Formal Definition Core

Graph

A graph G = (V, E) consists of a finite non-empty set V of vertices (nodes) and a set E of edges — pairs of vertices. In an undirected graph, edges are unordered pairs {u,v}. In a directed graph (digraph), edges are ordered pairs (u,v) with a direction from u to v.

1.2   Handshaking Lemma & Degree Sequence GATE Staple

Handshaking Lemma

In any undirected graph G = (V,E): Σ deg(v) = 2|E|
The sum of all vertex degrees equals twice the number of edges. Every edge contributes exactly 2 to the total degree sum — one for each endpoint.

Handshaking Lemma for Digraphs

Σ deg⁺(v) = Σ deg⁻(v) = |E|
Sum of all out-degrees = sum of all in-degrees = number of edges.

NAT pattern: A graph has 6 vertices, each with degree 4. How many edges does it have?

Σ deg = 6 × 4 = 24 = 2|E| → |E| = 12.

Can a graph have exactly 5 vertices each with odd degree?

Σ deg = 5 odd numbers. Sum of 5 odd numbers is odd. But Σ deg = 2|E| is always even. Contradiction — impossible. An even number of vertices must have odd degree.

Corollary — Number of Odd-Degree Vertices is Always Even

Since Σ deg = 2|E| is even, and even + even = even, the sum of odd-degree vertices' contributions must be even — which means there must be an even number of odd-degree vertices. Zero, two, four, six — never an odd count. GATE tests this as a direct True/False question.

Degree Sequence

The degree sequence of a graph is the list of vertex degrees arranged in non-decreasing (or non-increasing) order. A sequence of non-negative integers is graphical (can be the degree sequence of a simple graph) if and only if its sum is even and the Erdős–Gallai theorem conditions hold.

Is (3, 3, 2, 2, 2) a valid degree sequence?

Sum = 3+3+2+2+2 = 12, which is even. ✓ (Necessary condition satisfied.)

The complete graph K₄ has degree sequence (3,3,3,3). Adding an isolated vertex gives (3,3,3,3,0). This is not (3,3,2,2,2). Let's try to construct it: 5 vertices, two of degree 3, three of degree 2. This is the cycle C₅ with no extra edges — wait, C₅ has all degrees 2. A valid graph exists: take C₅ and add one chord. Actually the Petersen-style construction works — this sequence IS graphical.

The adjacency matrix of an undirected graph is always symmetric. For a directed graph, M[i][j] = 1 means edge from i to j but M[j][i] may be 0. The trace (diagonal sum) counts self-loops — zero for simple graphs.

Matrix Powers and Path Counting

The (i,j) entry of A^k (the k-th power of the adjacency matrix) gives the number of walks of length exactly k from vertex i to vertex j. This is a direct GATE question: "How many walks of length 2 exist from vertex 1 to vertex 3?" Compute A² and read entry [1][3].

K₅ properties: 5 vertices, 5×4/2 = 10 edges, every vertex has degree 4. It is 4-regular, simple, and complete. It also plays a starring role in planarity theory (Kuratowski's theorem).

Bipartite Graphs

Bipartite Graph

A graph G = (V,E) is bipartite if V can be partitioned into two disjoint sets V₁ and V₂ such that every edge connects a vertex in V₁ to a vertex in V₂. No edge exists within V₁ or within V₂.

Bipartite Characterization Theorem

A graph is bipartite if and only if it contains no odd-length cycles. This is a direct GATE theorem — tested as "which of these is bipartite?" Check for odd cycles.

C₄ (4-cycle) is bipartite: {1,3} and {2,4} form the bipartition. C₃ (triangle) is NOT bipartite — it is an odd cycle of length 3.

Trees are always bipartite — they have no cycles at all (and hence no odd cycles).

Walk vs Trail vs Path — Hierarchy

Every path is a trail. Every trail is a walk. The converses don't hold. When GATE says "shortest path" it means shortest walk with no repeated vertices (since repetitions only increase length). When asked about Euler circuits, the relevant structure is a circuit (closed trail — no repeated edges).

Connected Components

Connectivity

An undirected graph is connected if there is a path between every pair of vertices. A connected component is a maximal connected subgraph. A directed graph is strongly connected if there is a directed path from u to v AND from v to u for every pair u,v.

Cut Vertex and Bridge

A cut vertex (articulation point) is a vertex whose removal increases the number of connected components. A bridge is an edge whose removal increases the number of connected components.

Trees & Spanning Trees

3.1   Trees — Equivalent Definitions GATE Critical

A tree is one of the most important structures in discrete mathematics. These five statements are all equivalent — any one can serve as the definition, and the others follow as theorems:

Five Equivalent Definitions of a Tree

A graph T with n vertices is a tree if and only if any ONE of the following holds:
1. T is connected and has no cycles (acyclic connected graph).
2. T is connected and has exactly n−1 edges.
3. T has no cycles and has exactly n−1 edges.
4. T is connected and every edge is a bridge.
5. There is exactly one path between every pair of vertices.

GATE Trap — Two Properties Determine a Tree

Connected + acyclic = tree. Connected + n−1 edges = tree. Acyclic + n−1 edges = tree. But connected alone, or acyclic alone, or n−1 edges alone is NOT enough to guarantee a tree. GATE presents graphs and asks "is this a tree?" — always verify two properties.

3.2   Spanning Trees & Counting GATE Tested

Spanning Tree

A spanning tree of a connected graph G is a subgraph that is a tree and includes all vertices of G. It has exactly |V|−1 edges selected from E such that all vertices remain connected.

Kirchhoff's Matrix Tree Theorem

The number of distinct spanning trees of G equals any cofactor of the Laplacian matrix L = D − A, where D is the degree matrix (diagonal) and A is the adjacency matrix.

Number of spanning trees of Kₙ (complete graph on n vertices):

By Cayley's formula: the number of labeled spanning trees of Kₙ is nⁿ⁻².

K₁: 1¹⁻² = 1⁻¹ = 1 (just the single vertex) K₂: 2²⁻² = 2⁰ = 1 (single edge is the only spanning tree) K₃: 3³⁻² = 3¹ = 3 (three spanning trees of K₃) K₄: 4⁴⁻² = 4² = 16 (sixteen spanning trees of K₄) Kₙ: nⁿ⁻² spanning trees

Minimum Spanning Trees (MST)

For a weighted graph, the MST is the spanning tree with minimum total edge weight. Two classic algorithms:

Kruskal's Algorithm

Sort all edges by weight. Add edges greedily — add if it doesn't create a cycle. Use Union-Find to detect cycles. Time: O(E log E).

Prim's Algorithm

Start from any vertex. Greedily add the minimum weight edge connecting the current tree to a new vertex. Time: O(E log V) with priority queue.

Cut Property — Why Greedy Works for MST

For any cut (partition of vertices into two sets S and V−S), the minimum weight edge crossing the cut belongs to every MST. This is the mathematical guarantee behind both Prim's and Kruskal's algorithms — the greedy choice at each step is provably optimal.

Planarity & Graph Coloring

4.1   Planar Graphs & Euler's Formula GATE Critical

Planar Graph

A graph is planar if it can be drawn in the plane such that no two edges cross (except at shared vertices). The drawing is called a planar embedding. Planarity is a property of the graph, not a specific drawing.

Euler's Formula for Planar Graphs

For any connected planar graph: V − E + F = 2
where V = vertices, E = edges, F = faces (regions, including the unbounded outer face).

Verify Euler's formula on K₄:

K₄: V=4, E=6. Draw it as a triangle with a central vertex. F = 4 (3 inner triangular faces + 1 outer face). V−E+F = 4−6+4 = 2 ✓

Necessary Conditions for Planarity

Edge Bound for Planar Graphs

For a simple connected planar graph with V≥3 vertices:
E ≤ 3V − 6 (general)
E ≤ 2V − 4 (if the graph has no triangles — no cycles of length 3)

Prove K₅ is non-planar using the edge bound:

K₅ has V=5, E=10. Check: 3V−6 = 3(5)−6 = 9. But E=10 > 9. Violates the bound. Therefore K₅ is non-planar. ✓

Prove K₃,₃ is non-planar:

K₃,₃ has V=6, E=9. K₃,₃ is bipartite so has no triangles (odd cycles). Use: E ≤ 2V−4 = 8. But E=9 > 8. Non-planar. ✓

Kuratowski's Theorem

A graph is planar if and only if it contains no subgraph that is a subdivision of K₅ or K₃,₃. A subdivision replaces edges with paths (adds intermediate vertices). This is both necessary and sufficient — the complete characterization.

E ≤ 3V−6 is Necessary, Not Sufficient

A graph satisfying E ≤ 3V−6 is not necessarily planar — it just hasn't been ruled out by this condition. To prove planarity, you must exhibit a planar drawing. To prove non-planarity, use edge bounds (sufficient for ruling out), Kuratowski's theorem, or direct argument.

4.2   Graph Coloring GATE Tested

Proper Coloring & Chromatic Number

A proper k-coloring assigns one of k colors to each vertex such that no two adjacent vertices have the same color. The chromatic number χ(G) is the minimum k for which G has a proper coloring.

Four Color Theorem

Every planar graph can be properly colored with at most 4 colors. Equivalently, every map can be colored with 4 colors such that no two adjacent regions share a color. Proved by Appel and Haken in 1976 using computer assistance — the first major theorem proved this way.

Chromatic Number Bounds

Lower bound: χ(G) ≥ ω(G) where ω(G) is the clique number (size of the largest complete subgraph). Every Kₖ subgraph needs k colors.
Upper bound: χ(G) ≤ Δ(G) + 1 where Δ(G) is the maximum degree (greedy coloring). Brooks' theorem tightens this: χ(G) ≤ Δ(G) unless G is a complete graph or odd cycle.

Chapter 05
Euler & Hamiltonian Paths

5.1   Euler Paths & Circuits GATE Tested

Euler Path & Euler Circuit

An Euler path is a trail (no repeated edges) that visits every edge of the graph exactly once. An Euler circuit is an Euler path that starts and ends at the same vertex.

Euler's Theorems — Exact Conditions

Euler Circuit exists ↔ Graph is connected AND every vertex has even degree.
Euler Path exists (non-circuit) ↔ Graph is connected AND exactly two vertices have odd degree (path starts at one, ends at the other).
If more than 2 vertices have odd degree → no Euler path at all.

The Seven Bridges of Königsberg: The city had 4 landmasses and 7 bridges. Does an Euler circuit exist?

Model as a graph: 4 vertices (landmasses), 7 edges (bridges). Degrees: 3, 3, 3, 5 — ALL odd. More than 2 odd-degree vertices → no Euler path exists. Euler proved this in 1736, founding graph theory.

NAT pattern: In a graph where exactly 2 vertices have odd degree, how many Euler paths exist? At least 1 (start at one odd-degree vertex, end at the other). The exact count requires the BEST algorithm.

Euler Circuit vs Hamiltonian Circuit — Easily Confused

Euler circuit visits every EDGE exactly once. Hamiltonian circuit visits every VERTEX exactly once. The conditions for Euler circuits are clean and checkable in polynomial time. Determining whether a Hamiltonian circuit exists is NP-complete — no efficient algorithm is known.

5.2   Hamiltonian Paths & Circuits GATE Tested

Hamiltonian Path & Circuit

A Hamiltonian path visits every vertex exactly once. A Hamiltonian circuit is a Hamiltonian path that returns to the starting vertex. Unlike Euler, there is no simple necessary and sufficient condition.

Sufficient Conditions (not necessary)

TheoremConditionGuarantees
Dirac's TheoremG has n≥3 vertices; every vertex has degree ≥ n/2Hamiltonian circuit exists
Ore's TheoremFor every non-adjacent pair u,v: deg(u)+deg(v) ≥ nHamiltonian circuit exists

These are sufficient but not necessary — a graph can have a Hamiltonian circuit even if the conditions don't hold. And a graph can fail both conditions without having one.

Kₙ has Hamiltonian circuits; Kₘ,ₙ has one iff m=n

The complete graph Kₙ has (n−1)!/2 distinct Hamiltonian circuits (fix one vertex, permute the rest, divide by 2 for direction). The complete bipartite graph Kₘ,ₙ has a Hamiltonian circuit if and only if m = n ≥ 2 — you must alternate between the two sets, so equal sizes are required.

Chapter 06
Counting Principles — Sum, Product, Inclusion-Exclusion

6.1   The Fundamental Counting Rules Core

Sum Rule (Rule of Addition)

If task A can be done in m ways and task B can be done in n ways, and the tasks are mutually exclusive (doing one rules out the other), then either A or B can be done in m + n ways.

Product Rule (Rule of Multiplication)

If task A can be done in m ways and, independently, task B can be done in n ways, then both A and B can be done (in sequence) in m × n ways.

Sum rule: A student can choose a programming language from {C, C++, Java} (3 options) or a scripting language from {Python, Ruby} (2 options). Total choices = 3 + 2 = 5.

Product rule: A password consists of 3 letters followed by 2 digits. Each letter: 26 choices. Each digit: 10 choices. Total passwords = 26 × 26 × 26 × 10 × 10 = 26³ × 10² = 1,757,600.

Inclusion-Exclusion Principle — Generalized

Inclusion-Exclusion for 3 Sets

|A ∪ B ∪ C| = |A| + |B| + |C| − |A∩B| − |A∩C| − |B∩C| + |A∩B∩C|

How many integers from 1 to 100 are divisible by 2, 3, or 5?

|A| = ⌊100/2⌋ = 50 (div by 2) |B| = ⌊100/3⌋ = 33 (div by 3) |C| = ⌊100/5⌋ = 20 (div by 5) |A∩B| = ⌊100/6⌋ = 16 (div by 6) |A∩C| = ⌊100/10⌋ = 10 (div by 10) |B∩C| = ⌊100/15⌋ = 6 (div by 15) |A∩B∩C| = ⌊100/30⌋ = 3 (div by 30) |A∪B∪C| = 50+33+20 − 16−10−6 + 3 = 74

Division Rule

Division Rule

If n objects are partitioned into groups of k identical objects each, the number of distinct groups = n/k. Used when counting overcounts by a factor of k. Example: counting unordered pairs from n elements = n(n−1)/2 because each pair is counted twice in n(n−1) ordered pairs.

7.1   Permutations — Ordered Arrangements GATE Tested

Permutation P(n,r)

The number of ways to arrange r objects chosen from n distinct objects where order matters: P(n,r) = n!/(n−r)! = n(n−1)(n−2)...(n−r+1)

Circular Permutations — Fix One Element

In circular permutations, rotations of the same arrangement are considered identical. Fix one person/object in place, arrange the remaining n−1 in (n−1)! ways. For necklaces (or cases where flipping is also identical), divide by 2: (n−1)!/2.

7.2   Combinations — Unordered Selections GATE Tested

Combination C(n,r)

The number of ways to choose r objects from n distinct objects where order does NOT matter: C(n,r) = n! / (r!(n−r)!) = P(n,r) / r!

Key Identities

Binomial Theorem

Binomial Theorem

(x + y)ⁿ = Σₖ₌₀ⁿ C(n,k) xᵏ yⁿ⁻ᵏ
The coefficient of xᵏyⁿ⁻ᵏ in the expansion of (x+y)ⁿ is C(n,k).

Coefficient of x³y⁵ in (x+y)⁸: C(8,3) = 56. (k=3, n=8)

Sum of all coefficients in (x+y)⁸: Set x=y=1: (1+1)⁸ = 2⁸ = 256 = Σ C(8,k).

Sum of coefficients of even powers in (x+y)ⁿ: Set x=1,y=−1 and x=1,y=1, average: 2ⁿ⁻¹.

Stars and Bars — Combinations with Repetition

Combinations with Repetition

The number of ways to choose r items from n types with repetition allowed (order doesn't matter) is C(n+r−1, r). Also counts non-negative integer solutions to x₁+x₂+...+xₙ=r.

Distribute 10 identical candies among 4 children (each can get 0 or more):

C(4+10−1, 10) = C(13,10) = C(13,3) = 286.

Number of non-negative integer solutions to x₁+x₂+x₃ = 7:

C(3+7−1, 7) = C(9,7) = C(9,2) = 36.

If each xᵢ ≥ 1 (positive solutions): Substitute yᵢ = xᵢ−1 ≥ 0. Then y₁+y₂+y₃ = 7−3 = 4. C(3+4−1,4) = C(6,4) = 15.

Pigeonhole Principle & Recurrence Relations

8.1   Pigeonhole Principle GATE Tested

Pigeonhole Principle

If n+1 or more objects are distributed into n boxes, then at least one box contains at least 2 objects.
Generalized: If N objects are distributed into k boxes, then at least one box contains at least ⌈N/k⌉ objects.

Among 13 people, at least 2 share the same birth month. (13 people, 12 months — 13 > 12.)

Among n+1 integers from {1,...,2n}, at least two are consecutive (differ by 1).
Pair them: {1,2}, {3,4}, ..., {2n−1,2n} — n pairs. With n+1 integers chosen from n pairs, at least one pair is fully represented → two consecutive integers.

In a graph with n≥2 vertices, at least two vertices have the same degree.
Possible degrees: 0,1,2,...,n−1 — seems like n choices for n vertices. But degree 0 (isolated vertex) and degree n−1 (connected to all) cannot both exist simultaneously. So only n−1 distinct degrees are possible for n vertices → at least two vertices have the same degree.

Applying Pigeonhole — The Template

Step 1: Identify the "pigeons" (objects). Step 2: Identify the "holes" (categories). Step 3: Show pigeons > holes, conclude some hole has ≥2 pigeons. The art is in choosing the right categorization — GATE sometimes requires a non-obvious partition.

8.2   Recurrence Relations GATE Tested

Recurrence Relation

An equation that defines each term of a sequence in terms of previous terms. A closed-form solution (or explicit formula) expresses the n-th term directly without reference to previous terms.

Linear Homogeneous Recurrences with Constant Coefficients

Form: aₙ = c₁aₙ₋₁ + c₂aₙ₋₂ + ... + cₖaₙ₋ₖ (no non-recurrence terms on the right side)

Characteristic Equation Method

Step 1: Write the characteristic equation by substituting aₙ = rⁿ: rⁿ = c₁rⁿ⁻¹ + c₂rⁿ⁻² + ... + cₖrⁿ⁻ᵏ → rᵏ = c₁rᵏ⁻¹ + ... + cₖ.
Step 2: Find the roots r₁, r₂, ..., rₖ of the characteristic polynomial.
Step 3: If all roots distinct: general solution = α₁r₁ⁿ + α₂r₂ⁿ + ... + αₖrₖⁿ.
If root r has multiplicity m: contributes (α₁ + α₂n + ... + αₘnᵐ⁻¹)rⁿ.
Step 4: Use initial conditions to solve for α₁, α₂, ..., αₖ.

Solve: aₙ = 5aₙ₋₁ − 6aₙ₋₂, with a₀=1, a₁=4

Characteristic equation: r² = 5r − 6 → r² − 5r + 6 = 0 Roots: (r−2)(r−3) = 0 → r₁=2, r₂=3 (distinct roots) General solution: aₙ = α₁·2ⁿ + α₂·3ⁿ Apply initial conditions: a₀ = α₁ + α₂ = 1 a₁ = 2α₁ + 3α₂ = 4 From first: α₁ = 1 − α₂ Substituting: 2(1−α₂) + 3α₂ = 4 → 2 + α₂ = 4 → α₂ = 2, α₁ = −1 Solution: aₙ = −1·2ⁿ + 2·3ⁿ = 2·3ⁿ − 2ⁿ

Fibonacci: Fₙ = Fₙ₋₁ + Fₙ₋₂, F₀=0, F₁=1

Characteristic equation: r² − r − 1 = 0 Roots: r = (1 ± √5)/2 φ = (1+√5)/2 ≈ 1.618 (golden ratio) ψ = (1−√5)/2 ≈ −0.618 General solution: Fₙ = α₁φⁿ + α₂ψⁿ Applying F₀=0, F₁=1: Fₙ = (φⁿ − ψⁿ)/√5 ← Binet's formula For large n: Fₙ ≈ φⁿ/√5 (since |ψ|<1, ψⁿ→0)

Non-Homogeneous Recurrences

Non-Homogeneous Form

aₙ = c₁aₙ₋₁ + ... + cₖaₙ₋ₖ + f(n) where f(n) ≠ 0.
Solution = homogeneous solution + particular solution.
Find the particular solution by guessing its form based on f(n):
f(n) = dⁿ → try pₙ = Adⁿ (or Anⁱdⁿ if d is a root of multiplicity i).
f(n) = polynomial of degree k → try polynomial of degree k.

Master Theorem (for divide-and-conquer recurrences)

Master Theorem

T(n) = aT(n/b) + f(n) where a≥1, b>1:
Let c = log_b(a).
Case 1: f(n) = O(nᶜ⁻ᵉ) for some ε>0 → T(n) = Θ(nᶜ)
Case 2: f(n) = Θ(nᶜ logᵏn) → T(n) = Θ(nᶜ logᵏ⁺¹n)
Case 3: f(n) = Ω(nᶜ⁺ᵉ) and af(n/b) ≤ cf(n) → T(n) = Θ(f(n))

Merge Sort: T(n) = 2T(n/2) + O(n). a=2, b=2, c=log₂2=1. f(n)=O(n)=O(n¹)=Θ(n¹log⁰n). Case 2 with k=0: T(n)=Θ(n log n). ✓

Binary Search: T(n) = T(n/2) + O(1). a=1, b=2, c=log₂1=0. f(n)=O(1)=Θ(n⁰log⁰n). Case 2 with k=0: T(n)=Θ(log n). ✓

Naive Matrix Multiply: T(n)=8T(n/2)+O(n²). a=8,b=2,c=log₂8=3. f(n)=O(n²)=O(n³⁻¹). Case 1: T(n)=Θ(n³). ✓

1Odd-degree vertex countThe number of vertices with odd degree is always even (Handshaking Lemma). If a question gives a degree sequence with an odd number of odd-degree vertices, no such graph exists.
2Tree: connected alone ≠ treeA tree requires BOTH connectivity AND acyclicity (equivalently, connected + exactly n−1 edges). Check two conditions.
3Spanning tree count for KₙCayley's formula: nⁿ⁻² spanning trees. K₃ has 3¹=3. K₄ has 4²=16. Very frequently tested as direct NAT.
4Euler's formula facesF includes the outer (unbounded) face. For a planar graph embedded in a plane, always count the infinite outer region as one face. V−E+F=2 always holds for connected planar graphs.
5K₅ and K₃,₃ non-planarityK₅: E=10 > 3(5)−6=9. K₃,₃: E=9 > 2(6)−4=8 (no triangles bound). Memorize both proofs — they appear directly in GATE.
6Euler circuit requires ALL even degreesAn Euler circuit exists iff connected + every vertex has even degree. Even ONE odd-degree vertex means no Euler circuit (only Euler path if exactly two odd-degree vertices, starting and ending at them).
7Hamiltonian vs EulerEuler = every EDGE exactly once (polynomial-time checkable). Hamiltonian = every VERTEX exactly once (NP-complete to check). Easy to confuse — the problems are very different in complexity.
8Chromatic number of bipartiteAny non-empty bipartite graph has χ=2. Trees are bipartite → χ=2. Odd cycles have χ=3. Even cycles have χ=2.
9Circular permutations(n−1)! not n!. Fix one element's position (rotations are identical), arrange remaining n−1 elements. For necklaces: additionally divide by 2 for reflections.
10Stars and bars formulaDistributing r identical items into n distinct boxes (0 allowed) = C(n+r−1, r). For positive integer solutions (each ≥1): substitute yᵢ=xᵢ−1 first, reducing r by n.
11C(n,r) vs P(n,r)P(n,r) = n!/(n−r)! for ordered. C(n,r) = P(n,r)/r! for unordered. "How many ways to select" → C. "How many ways to arrange" → P. The word "arrange" or "order" is the signal.
12Master theorem Case 2Case 2 applies when f(n) = Θ(nᶜ logᵏn). Result: Θ(nᶜ logᵏ⁺¹n). Most missed: the exponent of log increases by 1, not stays the same. T(n)=2T(n/2)+n gives Θ(n log n), not Θ(n).

9.2   Master Formula Reference

Graph Theory Σ deg(v) = 2|E| (Handshaking Lemma) |V| − |E| + |F| = 2 (Euler's formula for planar graphs) E ≤ 3V − 6 (simple planar, V≥3) (Planarity bound — triangles allowed) E ≤ 2V − 4 (simple planar, no triangles)(Planarity bound — bipartite) |E| = n(n−1)/2 for Kₙ (Complete graph edges) |E| = mn for Kₘ,ₙ (Complete bipartite edges) Spanning trees of Kₙ = nⁿ⁻² (Cayley's formula) Tree: |E| = |V| − 1 (Tree edge count)
Combinatorics P(n,r) = n!/(n−r)! (Ordered selection) C(n,r) = n!/(r!(n−r)!) (Unordered selection) C(n,r) = C(n−1,r−1) + C(n−1,r) (Pascal's identity) (x+y)ⁿ = Σ C(n,k) xᵏyⁿ⁻ᵏ (Binomial theorem) Σₖ C(n,k) = 2ⁿ (Sum of binomial coefficients) Repetition combinations: C(n+r−1, r) (Stars and bars) Circular permutations: (n−1)! (Without reflection) Necklace permutations: (n−1)!/2 (With reflection symmetry) Multinomial: n!/(n₁!n₂!...nₖ!) (Arrange with identical elements)

9.3   GATE Question Patterns

  • Handshaking NAT: "A graph has 7 vertices each of degree 3 — how many edges?" Σdeg=21 is odd — impossible. No such graph exists. 1 mark.
  • Spanning tree count: "How many spanning trees does K₅ have?" Cayley: 5³ = 125. Direct 1-mark NAT.
  • Planarity check: "Is K₃,₄ planar?" E=12, V=7. 3V−6=15. E≤15 ✓. But K₃,₄ has no triangles: 2V−4=10. E=12>10. Not planar. 2-mark reasoning.
  • Chromatic number: "What is χ(C₇)?" Odd cycle → 3. "What is χ(K₄)?" Complete → 4. 1-mark MCQ.
  • Euler circuit existence: "Does graph G have an Euler circuit?" Count odd-degree vertices. If zero → yes. If any → no. 1 mark.
  • Bipartite check: "Is the graph with this adjacency matrix bipartite?" Check for odd cycles by BFS 2-coloring. 1-2 marks.
  • Combination counting NAT: "Number of ways to distribute 8 identical balls into 3 distinct boxes with each box ≥2." Stars and bars after substitution: C(5+3−1,5)=C(7,5)=21. 2 marks.
  • Recurrence solving: "Solve T(n)=3T(n/3)+n." a=3,b=3,c=1,f(n)=n=Θ(n¹). Case 2: T(n)=Θ(n log n). 1-2 marks.
  • Pigeonhole application: "What is the minimum number of students needed to guarantee 3 with the same birth month?" ⌈N/12⌉=3 → N=25. 1 mark.
  • Binomial coefficient: "Coefficient of x⁴ in (1+x)¹⁰?" C(10,4)=210. 1 mark.
