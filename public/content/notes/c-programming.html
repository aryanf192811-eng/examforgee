Chapter 01
Basics — Variables, Data Types & I/O

1.1   Why C? The Mental Model Beginner

C is a thin layer above the hardware. When you write int x = 5, you are literally asking the CPU to reserve 4 bytes of memory and store the bit pattern for 5 there. No magic, no runtime, no garbage collector. You control memory. You control addresses. That directness is why C is hard — and why mastering it makes every other language feel easy.

The C Compilation Pipeline

Source.c → Preprocessor (handles #include, #define) → Source.i → Compiler (generates assembly) → Source.s → Assembler (generates object code) → Source.o → Linker (combines with libraries) → Executable.
Understanding this explains why #include "stdio.h" adds I/O functions, why #define is textual substitution (not a variable), and why linking errors are different from compilation errors.

Integer Overflow — Silent and Deadly

If an int holds 2147483647 (INT_MAX) and you add 1, it wraps to −2147483648. This is undefined behaviour in C — the compiler may do anything. In practice on most systems it wraps, but GATE questions that ask "what is the output" when overflow occurs are testing whether you know the value wraps. Always check the range of the type against the values being stored.

Short-Circuit Evaluation — Very Common in GATE

In A && B: if A is false, B is NEVER evaluated. In A || B: if A is true, B is NEVER evaluated. This means side effects in B (like i++) may not happen. GATE output prediction questions exploit this: if (x != 0 && ++y > 0) — if x is 0, y is never incremented.

Chapter 04
Functions & Recursion

4.1   Function Mechanics — Call Stack & Pass by Value GATE Critical

Call by Value — C Only Has This

C passes ALL arguments by value — a copy of the argument is made and passed to the function. Modifying the parameter inside the function does NOT affect the original variable. To modify the original, you must pass its address (a pointer).

void swap(int a, int b) { int t = a; a = b; b = t; // This swaps COPIES. Original x,y unchanged. } int main() { int x = 3, y = 7; swap(x, y); printf("%d %d\n", x, y); }
Output: 3 7 (swap did nothing to x and y)

Correct swap using pointers:

void swap(int *a, int *b) { int t = *a; *a = *b; *b = t; } swap(&x, &y); // Pass addresses

Recursion — The Call Stack Model

How Recursion Works in Memory

Each function call creates a new stack frame on the call stack — its own copy of parameters and local variables. When the function returns, its frame is destroyed. Recursive calls keep pushing frames until the base case is hit, then unwind. GATE tests: trace the call sequence, identify base case, compute the return value at each level.

int fact(int n) { if (n == 0) return 1; // base case return n * fact(n - 1); // recursive case } printf("%d\n", fact(4));
Trace: fact(4)=4×fact(3)=4×3×fact(2)=4×3×2×fact(1)=4×3×2×1×fact(0)=4×3×2×1×1=24. Output: 24
void mystery(int n) { if (n == 0) return; mystery(n - 1); // recursive call BEFORE print printf("%d ", n); } mystery(4);
Output: 1 2 3 4 (prints on the WAY BACK UP — after recursion returns)
void mystery2(int n) { if (n == 0) return; printf("%d ", n); // print BEFORE recursive call mystery2(n - 1); } mystery2(4);
Output: 4 3 2 1 (prints on the way DOWN)
Recursive Output Direction — Quick Rule

Print before the recursive call → prints in forward order (top-down). Print after the recursive call → prints in reverse order (bottom-up, like a stack being unwound). This is the #1 GATE recursion output prediction pattern.

Chapter 05
Arrays & Strings

5.1   Arrays — Memory Layout & Pointer Equivalence GATE Critical

Arrays Are Contiguous — This Is Everything

An array int a[5] stores 5 integers in consecutive memory locations. a[i] is just syntactic sugar for *(a + i). The array name a decays to a pointer to a[0] in most contexts. This is why you can pass arrays to functions without specifying size — you're passing a pointer.

int a[5] = {10, 20, 30, 40, 50}; // These are all equivalent: printf("%d\n", a[2]); // 30 printf("%d\n", *(a+2)); // 30 printf("%d\n", 2[a]); // 30 (yes, this is valid C!) // 2D array — row-major storage int b[3][4]; // 3 rows, 4 columns // b[i][j] is at address: base + (i*4 + j)*sizeof(int) // All 12 integers stored consecutively in memory

Strings — Null-Terminated Character Arrays

Strings in C

A string is a char array terminated by a null character '\0' (value 0). "hello" occupies 6 bytes: 'h','e','l','l','o','\0'. strlen() returns the number of characters BEFORE the null — it does NOT count '\0'. sizeof on the array returns the total size INCLUDING '\0'.

char s[] = "hello"; printf("%lu\n", sizeof(s)); // 6 (includes \0) printf("%lu\n", strlen(s)); // 5 (excludes \0) printf("%c\n", s[1]); // e printf("%d\n", s[0] - 'a'); // 104-97=7 ('h'-'a')
Output: 6 / 5 / e / 7

Chapter 06
Pointers — The Core of C

6.1   Pointer Fundamentals Most Tested

Pointer Mental Model

A pointer is a variable that stores a memory address. That's it. int *p means: p is a variable that holds the address of an int. &x gives the address of x. *p gives the value at the address p holds (dereferencing). The type of the pointer tells the compiler how many bytes to read when dereferencing.

int x = 42; int *p = &x; // p holds the address of x printf("%d\n", x); // 42 — value of x printf("%p\n", p); // 0x... — address of x (some hex address) printf("%d\n", *p); // 42 — value AT the address p holds *p = 100; // modify x through pointer printf("%d\n", x); // 100 — x is now 100!

Pointer Arithmetic

Pointer Arithmetic Scales by Type Size

When you do p + 1 on an int*, the address increases by sizeof(int) = 4 bytes, NOT by 1. The compiler automatically scales by the pointed-to type's size. This is why arrays and pointers work together seamlessly — a[i] and *(a+i) are identical.

int a[] = {10, 20, 30}; int *p = a; // p points to a[0] printf("%d\n", *p); // 10 printf("%d\n", *(p+1)); // 20 p++; // p now points to a[1] printf("%d\n", *p); // 20 printf("%d\n", p[1]); // 30 (p[1] = *(p+1))
Output: 10 / 20 / 20 / 30

Pointer to Pointer, const Pointers, Void Pointers

// Pointer to pointer int x = 5; int *p = &x; int **pp = &p; printf("%d\n", **pp); // 5 // const variations — very commonly tested int y = 10; const int *cp = &y; // pointer to const int: *cp can't be changed, cp can int * const pc = &y; // const pointer to int: cp can't be changed, *pc can const int * const cpc = &y; // both constant // void pointer — generic pointer, must cast before dereferencing void *vp = &x; printf("%d\n", *(int*)vp); // cast to int* first

Function Pointers

// int (*fp)(int, int) — pointer to function taking 2 ints, returning int int add(int a, int b) { return a + b; } int mul(int a, int b) { return a * b; } int (*fp)(int, int); fp = add; printf("%d\n", fp(3, 4)); // 7 fp = mul; printf("%d\n", fp(3, 4)); // 12

qsort() — The Standard Callback Pattern

Why qsort() Uses a Function Pointer

The C standard library's qsort() doesn't know what type of data you're sorting. Instead of writing a sort for ints, floats, strings, and structs separately, it accepts a comparison function pointer. You supply the "how to compare" logic; qsort supplies the "when to compare" logic. This is the callback pattern — the foundation of generic programming in C.

// Signature: void qsort(void *base, size_t n, size_t size, // int (*compar)(const void*, const void*)) // Compare function for ascending int sort int cmp_int(const void *a, const void *b) { return (*(int*)a) - (*(int*)b); // Returns negative if a < b (a comes first) // Returns 0 if equal, positive if a > b } int arr[] = {5, 2, 8, 1, 9}; qsort(arr, 5, sizeof(int), cmp_int); // arr is now {1, 2, 5, 8, 9} // For DESCENDING order: just swap a and b in the return int cmp_desc(const void *a, const void *b) { return (*(int*)b) - (*(int*)a); } // Sort strings: strcmp already returns the right value int cmp_str(const void *a, const void *b) { return strcmp(*(char**)a, *(char**)b); }

6.2   The Spiral Rule — Reading Complex Declarations GATE Critical

The Clockwise Spiral Rule

C declaration syntax can create unreadable chains of *, (), and []. The spiral rule cuts through this:
Start at the variable name. Spiral clockwise reading tokens in this priority order: (1) [], (2) (), (3) *.
In English: read right first, then left, alternating until all tokens are consumed.

Step-by-step decoding of 5 declarations — from simple to nightmare:

// Level 1: int *p; // Start at p → left: * → left: int // p is a POINTER TO int // Level 2: int *a[5]; // Start at a → right: [5] → left: * → left: int // a is an ARRAY OF 5 POINTERS TO int // Level 3: int (*a)[5]; // Start at a → left: * (parentheses group) → right: [5] → left: int // a is a POINTER TO AN ARRAY OF 5 ints // Level 4: int *(*fp)(int *, int); // Start at fp → left: * → right: (int*, int) → left: * → left: int // fp is a POINTER TO A FUNCTION TAKING (int*, int) RETURNING int* // Level 5 (nightmare): int (*(*fp)(int *))[10]; // Start at fp → left: * → right: (int*) → left: * → right: [10] → left: int // fp is a POINTER TO A FUNCTION TAKING (int*) // RETURNING A POINTER TO AN ARRAY OF 10 ints

The practical shortcut — typedef from inside out: For extremely complex types, build the declaration with typedefs from inside out. Name each layer and compose.
typedef int Row[10]; — Row is array of 10 ints.
typedef Row* (*FP)(int*); — FP is pointer to function(int*) returning Row*.
This is how production code handles it. The spiral rule is for reading others' code.

Structures, Unions & Enums

7.1   struct — Grouping Related Data GATE Tested

struct Point { int x, y; }; struct Point p1 = {3, 4}; p1.x = 10; // access via dot struct Point *ptr = &p1; ptr->y = 20; // access via arrow (= (*ptr).y) // typedef — cleaner syntax typedef struct { char name[50]; int age; float gpa; } Student; Student s = {"Aryan", 19, 9.2};
Structure Padding and Alignment

The compiler may insert padding bytes between struct members to align them to their natural alignment (e.g., int needs 4-byte alignment). sizeof(struct) may be larger than the sum of member sizes. Rule of thumb: members are padded so each starts at a multiple of its own size. Total struct size is padded to the size of the largest member.
Example: struct{char c; int n;} — char at offset 0, 3 padding bytes, int at offset 4. sizeof = 8, not 5.

Union — Shared Memory

Union vs Struct

A union allocates memory for its largest member only — all members share the same memory location. Writing one member and reading another gives the raw bytes of what was written, interpreted as the other type. sizeof(union) = sizeof(largest member) + any padding. Used for type punning and memory-efficient data representation.

union U { int i; char c; }; union U u; u.i = 65; printf("%c\n", u.c); // reads same memory as char printf("%lu\n", sizeof(u));
Output: A / 4 (65 in ASCII is 'A'; size = sizeof(int) = 4)

__attribute__((packed)) — Defeating Alignment

__attribute__((packed))

A GCC extension that tells the compiler to pack all struct members together with NO padding — even if it causes misalignment. sizeof(packed struct) = exact sum of member sizes.
Why use it: Network protocol headers, binary file formats, hardware registers where exact byte layout is mandated.
Cost: Misaligned reads/writes are slower on x86 and can cause bus errors on strict-alignment architectures (ARM in older modes). Never use in performance-critical code.

struct __attribute__((__packed__)) Packed { char a; // offset 0 (1 byte) int b; // offset 1 (4 bytes) ← normally would be offset 4 char c; // offset 5 (1 byte) }; // sizeof(Packed) = 6 (not 12 as normal struct would give) struct Normal { char a; int b; char c; }; // sizeof(Normal) = 12: a(1)+pad(3)+b(4)+c(1)+pad(3) = 12
Strict Aliasing — The Invisible UB

C's strict aliasing rule states: you may NOT access an object through a pointer of a different type (except char*). Doing so is undefined behaviour — the compiler may assume it never happens and optimize away safety checks.
float f = 3.14f; int i = *(int*)&f; — This is UB. The compiler is allowed to assume the int* and float* never alias the same memory.
The SAFE way to reinterpret bytes: use memcpy or a union. Both are defined. GCC's -fno-strict-aliasing disables this rule, but relying on it is non-portable.

Four Classic Memory Bugs — GATE Output Traps

Memory leak: malloc without free. Memory used but never returned to OS. Program still runs but accumulates wasted memory.
Dangling pointer: Pointer used after free(). Undefined behaviour — may work, may crash, may corrupt.
Double free: free() called twice on same pointer. Undefined behaviour — corrupts heap.
Buffer overflow: Writing beyond allocated bounds. Silently corrupts adjacent memory — hardest bug to find.

8.2   Endianness — Byte Order in Memory GATE Tested

Little-Endian vs Big-Endian

When a multi-byte value (like a 4-byte int) is stored in memory, the bytes can be arranged in two ways:
Little-Endian (LE): Least significant byte (LSB) stored at the lowest memory address. Used by x86, x86-64 (Intel, AMD) — the vast majority of modern computers.
Big-Endian (BE): Most significant byte (MSB) at the lowest address. Used by network protocols (IP, TCP header byte order), SPARC, some ARM modes.
The byte ORDER changes; the bit order within each byte does not.

How is 0x12345678 stored at address 0x1000?

Value: 0x12345678 (32-bit integer) Bytes: 0x12, 0x34, 0x56, 0x78 Memory address: 0x1000 0x1001 0x1002 0x1003 ────── ────── ────── ────── Little-Endian: 0x78 0x56 0x34 0x12 (LSB first) Big-Endian: 0x12 0x34 0x56 0x78 (MSB first) Mnemonic: Little-Endian puts the LITTLE end (least significant) first.

Classic GATE output prediction — endianness via char* cast:

int x = 0x01000000; char *p = (char*)&x; printf("%d\n", *p); // print FIRST byte at lowest address
On Little-Endian (x86): first byte = 0x00 = 0. On Big-Endian: first byte = 0x01 = 1.
GATE assumes Little-Endian unless specified. Answer: 0

Check endianness at runtime:

int check_endian() { int x = 1; return *(char*)&x; // returns 1 on LE (LSB=1 is first), 0 on BE } // If first byte of integer 1 is 1 → little-endian // If first byte of integer 1 is 0 → big-endian (the 1 is in the last byte)
On Little-Endian: returns 1. On Big-Endian: returns 0.

Network byte order = Big-Endian: Network protocols store multi-byte integers in Big-Endian order (MSB first), regardless of the host system. POSIX provides htons() (host to network short), htonl(), ntohs(), ntohl() to convert between host and network byte order. If you see these functions in GATE code, they're handling endianness conversion.

Macro Side Effects — Classic GATE Trap

#define SQUARE(x) (x)*(x). What is SQUARE(++a)? Expands to (++a)*(++a) — increments a TWICE. This is undefined behaviour and a classic exam trap. Always use inline functions or typed constants instead of macros for computations involving variables that may have side effects.

Token Pasting (##) and Stringification (#) — Preprocessor Black Magic

# — Stringification Operator

The # operator before a macro parameter turns the parameter into a string literal at compile time. It converts the token as-is, before any other expansion. Used for debugging macros that print both the expression and its value.

#define STRINGIFY(x) #x #define PRINT_VAR(x) printf(#x " = %d\n", x) printf("%s\n", STRINGIFY(42)); // expands to printf("%s\n", "42") printf("%s\n", STRINGIFY(hello)); // expands to printf("%s\n", "hello") int count = 5; PRINT_VAR(count); // expands to printf("count" " = %d\n", count)
Output: 42 / hello / count = 5
## — Token Pasting Operator

The ## operator concatenates two tokens into a single new token during preprocessing. Used to generate variable names, function names, or struct field names programmatically. The result must be a valid C token.

#define CONCAT(a, b) a##b #define MAKE_VAR(n) var##n int CONCAT(x, 10) = 42; // becomes: int x10 = 42; int MAKE_VAR(1) = 100; // becomes: int var1 = 100; int MAKE_VAR(2) = 200; // becomes: int var2 = 200; printf("%d %d\n", var1, var2); // Practical use: generating struct field accessors #define GET(type, field) type##_get_##field // GET(student, name) → student_get_name (as a function name)
Output: 100 200

GATE output prediction with ##: The key insight is that ## happens BEFORE the rest of compilation. CONCAT(var, __LINE__) creates a variable name like var42 (where 42 is the line number). In output questions involving ##, first do all textual substitutions, THEN read the resulting C code as normal. Never try to evaluate ## and C semantics simultaneously.

10.1   Essential Patterns Every C Programmer Must Know Code Heavy

These are the building blocks. Once you internalize these patterns, you'll write and read C fluently. Each one has a specific context where it is the correct idiomatic solution.

Pattern 1 — Swap Two Variables

// Method 1: Temporary variable (safest, most readable) int t = a; a = b; b = t; // Method 2: XOR swap (no extra variable — avoid in production) a ^= b; b ^= a; a ^= b; // Warning: fails if a and b are the SAME variable! // Method 3: Arithmetic (can overflow) a = a + b; b = a - b; a = a - b;

Pattern 2 — Reverse an Array

void reverse(int a[], int n) { int lo = 0, hi = n - 1; while (lo < hi) { int t = a[lo]; a[lo] = a[hi]; a[hi] = t; lo++; hi--; } } // Time: O(n), Space: O(1)

Pattern 3 — Binary Search

int bsearch(int a[], int n, int key) { int lo = 0, hi = n - 1; while (lo <= hi) { int mid = lo + (hi - lo) / 2; // Avoid overflow: NOT (lo+hi)/2 if (a[mid] == key) return mid; else if (a[mid] < key) lo = mid + 1; else hi = mid - 1; } return -1; // not found } // Time: O(log n)

Pattern 4 — Linked List Operations

typedef struct Node { int data; struct Node *next; } Node; // Insert at head Node* insert_head(Node *head, int val) { Node *n = (Node*) malloc(sizeof(Node)); n->data = val; n->next = head; return n; } // Traverse and print void print_list(Node *head) { while (head) { printf("%d ", head->data); head = head->next; } } // Reverse a linked list (iterative) Node* reverse(Node *head) { Node *prev = NULL, *curr = head, *next; while (curr) { next = curr->next; curr->next = prev; prev = curr; curr = next; } return prev; }

Pattern 5 — Stack Using Array

#define MAXN 100 int stack[MAXN], top = -1; void push(int x) { stack[++top] = x; } int pop() { return stack[top--]; } int peek() { return stack[top]; } int empty() { return top == -1; }

Pattern 6 — GCD and LCM

int gcd(int a, int b) { return b == 0 ? a : gcd(b, a % b); } int lcm(int a, int b) { return a / gcd(a, b) * b; // divide first to avoid overflow }

Pattern 7 — Prime Sieve (Sieve of Eratosthenes)

void sieve(int n) { int is_prime[n+1]; for (int i = 0; i <= n; i++) is_prime[i] = 1; is_prime[0] = is_prime[1] = 0; for (int i = 2; i * i <= n; i++) { if (is_prime[i]) for (int j = i*i; j <= n; j += i) is_prime[j] = 0; } for (int i = 2; i <= n; i++) if (is_prime[i]) printf("%d ", i); } // Time: O(n log log n), Space: O(n)

Pattern 8 — Sorting (Bubble, Selection, Insertion)

// Bubble Sort — O(n²) worst/avg, O(n) best (already sorted) void bubble(int a[], int n) { for (int i = 0; i < n-1; i++) for (int j = 0; j < n-i-1; j++) if (a[j] > a[j+1]) { int t=a[j]; a[j]=a[j+1]; a[j+1]=t; } } // Insertion Sort — O(n²) worst, stable, good for small/nearly-sorted void insertion(int a[], int n) { for (int i = 1; i < n; i++) { int key = a[i], j = i - 1; while (j >= 0 && a[j] > key) { a[j+1] = a[j]; j--; } a[j+1] = key; } }

Pattern 9 — Fibonacci (Iterative vs Recursive)

// Recursive — O(2ⁿ) time (exponential!) int fib_rec(int n) { if (n <= 1) return n; return fib_rec(n-1) + fib_rec(n-2); } // Iterative — O(n) time, O(1) space int fib(int n) { if (n <= 1) return n; int a = 0, b = 1; for (int i = 2; i <= n; i++) { int c = a + b; a = b; b = c; } return b; }

Pattern 10 — Matrix Multiplication

void matmul(int A[][N], int B[][N], int C[][N], int n) { for (int i = 0; i < n; i++) for (int j = 0; j < n; j++) { C[i][j] = 0; for (int k = 0; k < n; k++) C[i][j] += A[i][k] * B[k][j]; } } // Time: O(n³)

GATE Traps & Output Predictions

11.1   Output Prediction — 15 Classic GATE Patterns

P1 — Comma operator:

int a = (1, 2, 3); // comma evaluates left to right, returns last printf("%d\n", a);
Output: 3

P2 — Pointer to array vs array of pointers:

int a[5] = {1,2,3,4,5}; int *p = a; printf("%d %d\n", *p, *(p+2)); p += 3; printf("%d\n", *p);
Output: 1 3 / 4

P3 — sizeof array vs pointer:

int a[10]; int *p = a; printf("%lu %lu\n", sizeof(a), sizeof(p));
Output: 40 8 (sizeof array = 10×4=40; sizeof pointer = 8 on 64-bit)

P4 — Integer division and modulo with negatives:

printf("%d %d\n", -7 / 2, -7 % 2);
Output: -3 -1 (C truncates toward zero: -7/2 = -3.5 → -3; remainder = -7 - (-3×2) = -7+6 = -1)

P5 — String literal vs char array:

char *s = "hello"; // pointer to string literal (read-only!) char t[] = "hello"; // writable array copy // s[0] = 'H'; ← UNDEFINED BEHAVIOUR (may segfault) t[0] = 'H'; // OK printf("%s %s\n", s, t);
Output: hello Hello

P6 — Recursive call with printf order:

int fun(int n) { if (n == 4) return n; return 2 * fun(n + 1); } printf("%d\n", fun(1));
Trace: fun(1)=2×fun(2)=2×2×fun(3)=2×2×2×fun(4)=2×2×2×4=32. Output: 32

P7 — Global vs local variable shadowing:

int x = 10; // global void f() { int x = 20; // local shadows global printf("%d\n", x); } int main() { f(); printf("%d\n", x); }
Output: 20 / 10

P8 — scanf return value:

int a, b; int r = scanf("%d %d", &a, &b); // r = number of successfully matched items // r = 2 if both matched, 1 if only first, 0 if none, EOF if end
Concept: scanf returns count of successfully read items.

P9 — Unary minus and increment:

int a = 5; printf("%d\n", -a++); // postfix ++ binds tighter than unary - printf("%d\n", a);
Output: -5 / 6 (a++ returns 5, then negated to -5; then a becomes 6)

P10 — Array passed to function decays:

void f(int a[]) { printf("%lu\n", sizeof(a)); // a is a pointer here! } int a[10]; printf("%lu\n", sizeof(a)); // 40 f(a);
Output: 40 / 8 (inside f, 'a' is a pointer, sizeof = 8, NOT 40)

11.2   All the Traps in One Place

#TrapCorrect Understanding
1= vs == in if conditionif (x = 5) assigns 5 to x and always evaluates to true (5 is nonzero). if (x == 5) compares. This is a legal but almost always wrong statement — compilers warn but it compiles. Write if (5 == x) as a defensive habit (Yoda condition).
2Array index out of boundsC never checks array bounds. a[10] on a 10-element array accesses memory beyond the array — undefined behaviour. It may silently produce wrong output, corrupt memory, or crash. GATE won't ask you to reason about out-of-bounds output.
3Returning local variable addressReturning a pointer to a local variable is UB — the local's stack frame is destroyed on return, so the returned pointer is dangling. The output is unpredictable. GATE tests: "What does this print?" when returning &local_var → UB (cannot determine output).
4sizeof is an operator, not a functionsizeof evaluates at compile time (mostly). sizeof(a++) does NOT evaluate a++ — no side effect. int a=5; int s=sizeof(a++); printf("%d",a); → prints 5, not 6.
5char signednesschar may be signed or unsigned depending on implementation. signed char: range −128 to 127. unsigned char: 0 to 255. GATE assumes typical implementation (signed char) unless stated otherwise.
6Precedence of * in declarationsint *a, b declares a as int* and b as int (NOT both int*). To declare two pointers: int *a, *b. The * binds to the name, not the type.
7printf with %d for floatprintf("%d\n", 3.14) is UB — %d expects int, 3.14 is double. The output is garbage. GATE questions always use correct format specifiers — if they don't, the answer is UB.
8Macro expansion doubles side effects#define SQ(x) (x)*(x). SQ(a++) expands to (a++)*(a++) — a incremented twice, result undefined. Always use inline functions for computations.
9Short circuit skips evaluationif (x != 0 && 1/x > 0) — if x=0, the 1/x is never evaluated (no division by zero). GATE exploits this: "What is the value of y after: if (0 && (y=5)) {}"? y is unchanged because right side never evaluates.
10Null pointer dereferenceDereferencing NULL (*p when p=NULL) is undefined behaviour — typically a segmentation fault. malloc returns NULL on failure. Always check before dereferencing.
11String comparison with ==if (s1 == s2) compares ADDRESSES of the strings (pointer comparison), not their content. Use strcmp(s1, s2) == 0 to compare content.
12Implicit int return from mainIn C99+, main implicitly returns 0 if no return statement is present. In older standards, the return value is unspecified. GATE assumes C99: missing return in main = return 0.
