Chapter 01
BFS — Breadth First Search

1.1   Algorithm & Core Properties Core

BFS explores a graph level by level — it visits all vertices at distance d from the source before visiting any vertex at distance d+1. It uses a queue (FIFO) to manage the frontier of vertices to explore.

BFS(G, s): for each vertex u ≠ s: color[u]=WHITE, d[u]=∞, π[u]=NIL color[s]=GRAY, d[s]=0, π[s]=NIL Q = empty queue Enqueue(Q, s) while Q not empty: u = Dequeue(Q) for each v in Adj[u]: if color[v] == WHITE: color[v]=GRAY, d[v]=d[u]+1, π[v]=u Enqueue(Q, v) color[u] = BLACK // WHITE = undiscovered, GRAY = in queue, BLACK = fully processed Time: O(V+E) [adjacency list] or O(V²) [adjacency matrix]
BFS Tree & Shortest Path

BFS produces a BFS tree rooted at s. The path from s to any reachable vertex v in the BFS tree is a shortest path (fewest edges) in the unweighted graph. d[v] = shortest-path distance from s to v in terms of edge count.

BFS on graph: s=1, Adj: 1→{2,3}, 2→{4,5}, 3→{5}, 4→{}, 5→{}

Queue state (showing vertex dequeued and newly added): Start: Q=[1] Dequeue 1 → discover 2(d=1), 3(d=1) → Q=[2,3] Dequeue 2 → discover 4(d=2), 5(d=2) → Q=[3,4,5] Dequeue 3 → 5 already GRAY, skip → Q=[4,5] Dequeue 4 → no new neighbours → Q=[5] Dequeue 5 → no new neighbours → Q=[] BFS distances: d[1]=0, d[2]=1, d[3]=1, d[4]=2, d[5]=2

1.2   BFS Applications GATE Tested

  • Shortest path in unweighted graphs — by definition of BFS levels.
  • Testing bipartiteness — 2-colour the graph during BFS. If any edge connects two vertices of the same colour → not bipartite. If BFS completes without conflict → bipartite.
  • Connected components — run BFS from each unvisited vertex. Number of BFS initiations = number of components.
  • Level-order traversal of trees — BFS on a tree = level-order traversal.
  • Minimum number of steps — any problem reducible to finding shortest path in an implicit graph (word ladder, knight's tour, maze) → BFS.
BFS for Bipartiteness — The Connection

A graph is bipartite iff it has no odd cycles (from File 2 of DM). BFS detects this by attempting a 2-colouring: assign alternating colours at each level. If BFS finds an edge between two same-colour vertices (a cross edge at the same level or a back edge to the same level), an odd cycle exists → not bipartite. This runs in O(V+E).

Chapter 02
DFS — Depth First Search & Edge Classification

2.1   Algorithm, Timestamps & Edge Types GATE Critical

DFS explores as deep as possible before backtracking. It uses the call stack (recursion) or an explicit stack. DFS assigns timestamps to each vertex: d[v] (discovery time) and f[v] (finish time). These timestamps are the key to classifying edges and detecting properties.

DFS(G): for each u: color[u]=WHITE, π[u]=NIL time = 0 for each u: if color[u]==WHITE: DFS_Visit(G, u) DFS_Visit(G, u): time++; d[u]=time; color[u]=GRAY for each v in Adj[u]: if color[v]==WHITE: π[v]=u; DFS_Visit(G, v) color[u]=BLACK; time++; f[u]=time Time: Θ(V+E) [processes every vertex and edge exactly once]

Edge Classification in DFS

Four Edge Types (for directed graphs)

Tree edge: edge (u,v) where v was WHITE when first discovered. Forms the DFS tree.
Back edge: edge (u,v) where v is GRAY (ancestor of u in DFS tree). Creates a cycle.
Forward edge: edge (u,v) where v is BLACK and d[u] < d[v] (v is a descendant, already finished).
Cross edge: edge (u,v) where v is BLACK and d[u] > d[v] (v is in a different subtree or component).

Edge Typev's colour when edge (u,v) exploredTimestamp relation
Tree edgeWHITEd[u] < d[v] < f[v] < f[u]
Back edgeGRAYd[v] < d[u] < f[u] < f[v]
Forward edgeBLACKd[u] < d[v] < f[v] < f[u]
Cross edgeBLACKd[v] < f[v] < d[u] < f[u]
Undirected Graphs Have Only Tree and Back Edges

In an undirected graph, DFS produces only tree edges and back edges — no forward or cross edges. This is because every edge {u,v} is explored from both directions, and the second exploration always finds a non-WHITE vertex (either GRAY = back, but never forward or cross). GATE frequently tests: "In an undirected DFS, which edge types can occur?"

Parenthesis Theorem

For any two vertices u and v in DFS, exactly one of these holds:
(1) [d[u], f[u]] and [d[v], f[v]] are completely disjoint — u and v have no ancestor/descendant relationship.
(2) [d[u], f[u]] is completely inside [d[v], f[v]] — u is a descendant of v.
(3) [d[v], f[v]] is completely inside [d[u], f[u]] — v is a descendant of u.
Intervals never partially overlap.

2.2   DFS Properties & Applications GATE Tested

Cycle Detection

A directed graph has a cycle iff DFS finds a back edge. An undirected graph has a cycle iff DFS finds a back edge.

Topological Sort

If no back edges exist (DAG), output vertices in decreasing finish time order. Result is a topological ordering.

SCC Detection

Kosaraju's algorithm uses two DFS passes (on G and Gᵀ) to find strongly connected components.

Articulation Points

Vertices whose removal disconnects the graph. Found using DFS with low-link values.

Chapter 03
Topological Sort — Ordering DAGs

3.1   Definition & Two Algorithms GATE Critical

Topological Sort

A topological ordering of a directed acyclic graph (DAG) G=(V,E) is a linear ordering of all vertices such that if (u,v) ∈ E, then u appears before v. It represents a valid execution order for tasks with dependencies.

Topological Sort Requires a DAG

Topological sort is only defined for DAGs (directed acyclic graphs). If the graph has a cycle, no topological ordering exists — you can't linearize a cycle. Before running topological sort, either guarantee the graph is a DAG or check for cycles simultaneously. DFS-based topological sort detects cycles as back edges.

Algorithm 1 — DFS-Based (Finish Times)

Run DFS on the entire graph. After each vertex finishes (turns BLACK), prepend it to the output list. The resulting list is a topological ordering.

TopologicalSort_DFS(G): Run DFS(G), computing finish times f[v] As each vertex finishes, insert at FRONT of list L Return L // sorted by decreasing finish time Time: Θ(V+E) (dominated by DFS)

Algorithm 2 — Kahn's Algorithm (BFS-Based)

Repeatedly remove vertices with in-degree 0. Each removed vertex has no remaining dependencies — it can come first. Update in-degrees of neighbours after removal.

Kahn's Algorithm(G): Compute in-degree for all vertices Q = queue of all vertices with in-degree 0 count = 0 while Q not empty: u = Dequeue(Q) output u; count++ for each v in Adj[u]: in-degree[v]-- if in-degree[v] == 0: Enqueue(Q, v) if count < V: // CYCLE DETECTED report "Graph has a cycle, no topological ordering" Time: O(V+E)
Kahn's Algorithm Detects Cycles

If after Kahn's algorithm completes, the count of processed vertices is less than |V|, there must be a cycle — some vertices remain with non-zero in-degree because they depend on each other circularly. This dual purpose (sort + cycle detection) makes Kahn's practically useful.

Topological sort of DAG: edges 5→0, 5→2, 4→0, 4→1, 2→3, 3→1

In-degrees: 0→2, 1→2, 2→1, 3→1, 4→0, 5→0 Q = [4, 5] Dequeue 4 → output 4, update: 0(1), 1(1) → Q=[5] Dequeue 5 → output 5, update: 0(0), 2(0) → Q=[0,2] Dequeue 0 → output 0, no neighbours → Q=[2] Dequeue 2 → output 2, update: 3(0) → Q=[3] Dequeue 3 → output 3, update: 1(0) → Q=[1] Dequeue 1 → output 1 Result: 4, 5, 0, 2, 3, 1 ← one valid topological order (Multiple valid orderings may exist for the same DAG)

Chapter 04
Strongly Connected Components — Kosaraju's Algorithm

4.1   SCCs & Kosaraju's Algorithm GATE Tested

Strongly Connected Component (SCC)

A strongly connected component of a directed graph G is a maximal set of vertices C ⊆ V such that for every pair u, v ∈ C, there is a path from u to v AND a path from v to u. Every vertex is in exactly one SCC.

Kosaraju's Algorithm — Two-Pass DFS

Pass 1: Run DFS on G. Push each vertex onto a stack in order of finish times (last finished = top of stack).
Pass 2: Compute Gᵀ (transpose — reverse all edges). Run DFS on Gᵀ, processing vertices in stack order (highest finish time first). Each DFS tree in Pass 2 is one SCC.

Kosaraju(G): // Pass 1: DFS on G, record finish order Run DFS on G; push vertices onto stack S as they finish // Pass 2: DFS on Gᵀ in reverse finish order Compute Gᵀ (reverse all edges) while S not empty: u = pop(S) if u not visited: DFS_Visit(Gᵀ, u) // All vertices in this DFS tree = one SCC Time: Θ(V+E) (two DFS passes + transpose construction) Space: O(V+E)

Why It Works — The Key Insight

In Pass 1, the vertex with the highest finish time is in an SCC with no incoming edges from other SCCs (a "source SCC" in the condensation graph). Starting DFS from this vertex in Gᵀ visits exactly the vertices in its SCC — because in Gᵀ, this was a sink SCC, so DFS can't escape to other SCCs.

Condensation Graph

After finding all SCCs, contract each SCC to a single node. The resulting graph (the condensation) is always a DAG. This is fundamental: any directed graph, when contracted to its SCCs, becomes acyclic. GATE tests: "How many SCCs does this graph have?" and "Draw the condensation graph."

Tarjan's Algorithm — One-Pass Alternative

Tarjan's SCC algorithm finds all SCCs in a single DFS pass using a stack and low-link values (similar to bridge-finding). Same time complexity Θ(V+E). GATE focuses on Kosaraju's (conceptually cleaner), but knowing Tarjan's exists shows depth.

Chapter 05
Single-Source Shortest Paths — Dijkstra & Bellman-Ford

5.1   Setup — Relaxation & Optimal Substructure Core

Relaxation — The Core Operation

Every shortest-path algorithm uses relaxation. To relax edge (u,v) with weight w(u,v): if d[v] > d[u] + w(u,v), then update d[v] = d[u] + w(u,v) and set π[v] = u.
Initialize: d[s] = 0 for source, d[v] = ∞ for all others.

Optimal substructure: Any subpath of a shortest path is itself a shortest path. This is what allows both Dijkstra and Bellman-Ford to work — they build shortest paths incrementally.

Negative Edge Weights — When Standard Algorithms Fail

Dijkstra's algorithm assumes all edge weights are non-negative. A single negative edge can invalidate Dijkstra's greedy choice — a "later" path through a negative edge might be shorter than the path Dijkstra chose first. Bellman-Ford handles negative weights but not negative cycles. Floyd-Warshall can detect negative cycles.

5.2   Dijkstra's Algorithm Most Tested

Dijkstra's Algorithm — Greedy Single-Source Shortest Paths

Maintains a set S of vertices whose shortest paths are finalized. Repeatedly extracts the vertex u with minimum d[u] from the min-priority queue Q (not yet in S), adds it to S, and relaxes all edges out of u. Works correctly for non-negative edge weights.

Dijkstra(G, w, s): d[s]=0; d[v]=∞ for all v≠s; π[v]=NIL for all v Q = min-priority queue with all vertices, keyed by d[] S = {} while Q not empty: u = Extract-Min(Q) // O(log V) S = S ∪ {u} for each v in Adj[u]: if d[u] + w(u,v) < d[v]: d[v] = d[u] + w(u,v) π[v] = u Decrease-Key(Q, v, d[v]) // O(log V)

Binary HeapO(log V)O(log V)O((V+E) log V)Sparse graphs (E ≈ V)

Trace Dijkstra on weighted graph (source = A):

Graph: A→B(4), A→C(2), B→D(3), C→B(1), C→D(5) Initial: d[A]=0, d[B]=∞, d[C]=∞, d[D]=∞ Q = {A(0), B(∞), C(∞), D(∞)} Extract A(0): relax B→min(∞,0+4)=4, C→min(∞,0+2)=2 Q = {C(2), B(4), D(∞)} Extract C(2): relax B→min(4,2+1)=3, D→min(∞,2+5)=7 Q = {B(3), D(7)} Extract B(3): relax D→min(7,3+3)=6 Q = {D(6)} Extract D(6): no relaxation Final: d[A]=0, d[B]=3, d[C]=2, d[D]=6

Path to D: A→C→B→D (total weight 2+1+3=6). The direct path A→C→D costs 2+5=7 — longer. Dijkstra found the correct shortest path.

Dijkstra Fails on Negative Edges — The Classic Counterexample

Graph: A→B(2), A→C(4), B→C(−3). True shortest A→C = 2+(−3) = −1 (via B). Dijkstra extracts A first, relaxes B(2) and C(4). Extracts B(2), relaxes C to min(4,2−3)=−1 ✓ — this actually works in this case. But the key issue: Dijkstra may have already finalized C(4) before discovering B leads to a better path. With negative edges, once a vertex is extracted from Q, its distance is not guaranteed to be final.

5.3   Bellman-Ford Algorithm GATE Tested

Bellman-Ford — Handles Negative Weights

Relax ALL edges V−1 times. After k iterations, d[v] gives the shortest path distance using at most k edges. A shortest path with no cycles uses at most V−1 edges, so V−1 iterations suffice. A V-th iteration that still improves any distance indicates a negative cycle.

Bellman-Ford(G, w, s): d[s]=0; d[v]=∞; π[v]=NIL for all v≠s for i = 1 to V−1: for each edge (u,v) ∈ E: if d[u] + w(u,v) < d[v]: d[v] = d[u] + w(u,v) π[v] = u for each edge (u,v) ∈ E: // Negative cycle check if d[u] + w(u,v) < d[v]: return "NEGATIVE CYCLE DETECTED" return d[] Time: O(VE) Space: O(V)

Why V−1 iterations? A shortest path can have at most V−1 edges (otherwise it repeats a vertex, forming a cycle). In iteration i, Bellman-Ford correctly computes shortest paths using at most i edges. After V−1 iterations, all paths with at most V−1 edges are finalized.

Shortest Paths in DAGs — Topological Relaxation

For a DAG with any edge weights (even negative), we can find single-source shortest paths in O(V+E) — faster than both Dijkstra and Bellman-Ford:

DAG-Shortest-Paths(G, w, s): Topologically sort G Initialize d[s]=0, d[v]=∞ for others For each vertex u in topological order: for each v in Adj[u]: Relax(u, v, w) Time: O(V+E) — optimal for DAGs

All-Pairs Shortest Paths — Floyd-Warshall

6.1   Floyd-Warshall Algorithm GATE Critical

Floyd-Warshall — All-Pairs DP

Computes shortest paths between ALL pairs of vertices using dynamic programming. Handles negative edge weights. Detects negative cycles (diagonal becomes negative). Time: Θ(V³). Space: O(V²).

DP formulation: Let d^(k)[i][j] = shortest path from i to j using only vertices {1, 2, ..., k} as intermediate vertices.

Recurrence: d^(0)[i][j] = w(i,j) if edge exists, else ∞ (0 if i==j) d^(k)[i][j] = min( d^(k-1)[i][j], // don't use k d^(k-1)[i][k] + d^(k-1)[k][j] ) // use k as intermediate Floyd-Warshall(W): // W is n×n weight matrix D = W (copy, with D[i][i]=0) for k = 1 to n: for i = 1 to n: for j = 1 to n: D[i][j] = min(D[i][j], D[i][k] + D[k][j]) return D Time: Θ(n³) Space: O(n²) — in-place update possible

Trace Floyd-Warshall on:

Graph (directed, 3 vertices): 1→2: 3, 1→3: 8, 2→3: 1, 3→1: 2, 2→1: -4 Initial D⁽⁰⁾: 1 2 3 1 [ 0 3 8 ] 2 [-4 0 1 ] 3 [ 2 ∞ 0 ] k=1 (use vertex 1 as intermediate): D[2][3] = min(1, D[2][1]+D[1][3]) = min(1, -4+8) = 1 (no change) D[3][2] = min(∞, D[3][1]+D[1][2]) = min(∞, 2+3) = 5 D⁽¹⁾: same except [3][2] = 5 k=2 (use vertex 2): D[1][3] = min(8, D[1][2]+D[2][3]) = min(8, 3+1) = 4 D[3][3] = min(0, D[3][2]+D[2][3]) = min(0, 5+1) = 0 (no change) D⁽²⁾: D[1][3] = 4 k=3 (use vertex 3): D[1][2] = min(3, D[1][3]+D[3][2]) = min(3, 4+5) = 3 (no change) D[2][1] = min(-4, D[2][3]+D[3][1]) = min(-4, 1+2) = -4 (no change) Final: D[1][2]=3, D[1][3]=4, D[2][1]=-4, D[2][3]=1, D[3][1]=2, D[3][2]=5
Detecting Negative Cycles in Floyd-Warshall

After Floyd-Warshall completes, check the diagonal: if D[i][i] < 0 for any vertex i, then there is a negative-weight cycle through vertex i. A negative cycle means shortest paths are undefined (can keep going around the cycle to decrease path length indefinitely).

Chapter 07
Minimum Spanning Trees — Kruskal's & Prim's

7.1   MST — Theory & Greedy Correctness GATE Tested

Minimum Spanning Tree

A spanning tree of connected graph G is a subgraph that is a tree and includes all V vertices. Its weight = sum of edge weights. The MST is the spanning tree with minimum total weight. A graph may have multiple MSTs if edge weights are not distinct.

Cut Property — Why Greedy Works

For any cut (partition of V into two non-empty sets S and V−S), the minimum-weight edge crossing the cut is in EVERY MST of G (assuming unique weights). This guarantees both Kruskal's and Prim's greedy choices are always correct — at each step they add the cheapest edge that respects some cut.

7.2   Kruskal's Algorithm GATE Tested

Kruskal(G, w): Sort edges by weight: e₁ ≤ e₂ ≤ ... ≤ eₘ A = {} (MST edge set) for each vertex v: Make-Set(v) // Union-Find for each edge (u,v) in sorted order: if Find(u) ≠ Find(v): // u,v in different components A = A ∪ {(u,v)} Union(u, v) return A Time: O(E log E) = O(E log V) [sorting dominates] O(E · α(V)) [union-find operations, α ≈ constant] Total: O(E log V)
Union-Find (Disjoint Set Union)

Union-Find supports two operations: Find(x) = find the set containing x (returns representative), Union(x,y) = merge the sets containing x and y. With union-by-rank and path compression, each operation runs in O(α(n)) amortized time where α is the inverse Ackermann function — effectively constant (α(n) ≤ 4 for all practical n). GATE sometimes asks about the data structure used in Kruskal's.

Kruskal's on edges sorted by weight:

Edges sorted: (A,B,1), (B,C,2), (A,C,3), (B,D,4), (C,D,5) Vertices: A, B, C, D Add (A,B,1): A and B in different sets → add. A={A,B}, B={A,B} Add (B,C,2): B and C in different sets → add. {A,B,C} Add (A,C,3): A and C in SAME set → skip (would create cycle) Add (B,D,4): B and D in different sets → add. {A,B,C,D} MST edges: (A,B,1), (B,C,2), (B,D,4) Total weight: 7 All V−1=3 edges selected → done.

MST vs Shortest Path Tree — Different Problems

The MST minimizes total weight of the spanning tree. The shortest-path tree (from Dijkstra) minimizes distance from source to each vertex. These are different problems with different solutions. The MST is NOT a shortest-path tree in general, and the shortest-path tree is NOT an MST. GATE exploits this confusion.

  • BFS level order: "What is the BFS order starting from vertex s?" Draw the BFS tree level by level. When multiple vertices are at the same level, process them in adjacency list order. 1−2 marks.
  • DFS edge classification: "In DFS on this directed graph, classify edge (u,v) as tree/back/forward/cross." Check colour of v when edge is traversed. 1−2 marks.
  • Cycle detection: "Does this graph contain a cycle?" DFS − back edge exists? Or Kahn's − processed count < V? Direct 1-mark MCQ.
  • Topological ordering validity: "Which of the following is a valid topological ordering?" For each option, verify all edges go from earlier to later in the ordering. 1 mark.
  • Dijkstra trace: "What is the shortest distance from s to t?" Trace Dijkstra step by step showing the priority queue state at each extraction. Most common 2-mark question in this chapter.
  • Bellman-Ford negative cycle: "Does this graph have a negative cycle?" Run V iterations of Bellman-Ford — if V-th iteration still relaxes an edge, negative cycle exists. 2 marks.
  • Floyd-Warshall matrix: "What is D[i][j] after Floyd-Warshall?" Trace the DP table update. Negative diagonal = negative cycle. 2-mark NAT.
  • MST weight: "What is the weight of the MST of this graph?" Apply Kruskal's or Prim's — sort edges, add greedily avoiding cycles. Very common 1−2 mark question.
  • Number of MSTs: "How many distinct MSTs does this graph have?" Count the ways to choose among equal-weight edges at critical steps. Harder 2-mark question.
  • Algorithm applicability: "Which algorithm should be used to find shortest paths in a graph with negative edges but no negative cycles?" Bellman-Ford. "In an unweighted graph?" BFS. "In a DAG with negative weights?" DAG relaxation. 1-mark classification question.
